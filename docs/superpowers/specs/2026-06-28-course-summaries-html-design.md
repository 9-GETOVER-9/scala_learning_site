# 课程总结 HTML 资料馆设计

## 目标

将 `Scala/课程总结/` 的 21 篇总结和 `java/课程总结/` 的 19 篇总结生成可在 GitHub Pages 直接阅读的 HTML，并从首页和对应课程详情页提供入口。Markdown 继续作为唯一内容源。

## 信息架构

1. 新增 `summaries.html`，作为全部 40 篇课程总结的资料馆首页。
2. 索引页支持关键词搜索、Java/Scala 语言筛选和日期或课程编号浏览。
3. 生成结果保存为 `summaries/scala/<slug>.html` 和 `summaries/java/<slug>.html`。
4. 每篇阅读页包含语言、课程日期/编号、标题、正文、返回总结索引、上一篇和下一篇。
5. 首页顶部导航和主体内容增加“课程总结”入口。
6. 课程详情页在存在可靠映射时显示“阅读完整课程总结”。同一节课对应上/下两篇时显示两个链接。

## 生成流程

1. 新增 `scripts/build-summaries.ps1`，扫描两个课程总结目录。
2. 脚本从规范文件名中提取语言、日期/编号、标题和上/下标记。
3. Markdown 转换必须支持标题、段落、有序/无序列表、强调、行内代码、围栏代码块、引用、链接、分隔线和表格。
4. 脚本生成 40 份文章 HTML 及 `js/summaries-data.js` 元数据清单。
5. 输出必须可重复生成；重新运行不会制造重复记录或不稳定文件名。
6. 所有来自 Markdown 的文本在生成 HTML 时安全转义，不允许原始 HTML 或脚本注入页面。

## 课程映射

- Scala 课程优先按规范化后的日期与 `lesson.date` 匹配。
- Java 课程优先按文件名中的两位课程编号与课程数据中的课次匹配。
- 匹配结果写入元数据的 `lessonId`；无法确定时保持空值，只在总结索引出现，不在课程页生成可能错误的链接。
- 生成时输出匹配/未匹配数量，并将重复 slug、重复元数据 ID 或缺失源文件视为错误。

## 界面与可访问性

- 复用现有色彩、按钮、卡片和页面宽度，不引入新前端框架。
- 文章页使用适合长文阅读的行宽、行高、标题层级和代码滚动容器。
- 筛选控件使用真实 `label`，搜索结果数量通过可访问状态区域更新。
- 所有链接、按钮、标题和代码块保持键盘可用和语义化 HTML。
- 移动端使用单列卡片，代码和表格允许水平滚动，不撑破页面。

## Agency Agents 审查

实现后从官方 `msitarzewski/agency-agents` 仓库获取相关角色说明，并用于本地、只读审查。优先选用以下角色类型：

- 前端开发：语义结构、响应式、浏览器兼容性和维护性。
- UX/产品设计：入口可发现性、筛选流程和长文阅读体验。
- 可访问性：键盘、标题层级、表单标签、对比度和动态状态。
- 代码审查/现实校验：生成稳定性、内容安全、断链和过度设计。

审查输出一份按严重程度排序的 Markdown 报告。修复属于已批准范围的高严重度问题，再重新执行验证。

## 验证与完成标准

- 生成脚本成功产生 40 份非空 HTML，元数据数量也为 40。
- 每份生成 HTML 只引用存在的本地 CSS/JS/链接。
- 首页、总结索引、文章页和课程详情链接都可通过本地 HTTP 服务器访问。
- 搜索、语言筛选、空结果、返回索引、上/下一篇和课程直达链接通过浏览器检查。
- JavaScript 语法检查通过，并且现有首页筛选、课程详情和 Java 互动页无回归。
- Agency Agents 审查报告已生成，范围内高严重度问题已处理或明确记录。
