## 执行证据

- 课程总结生成：40/40 份非空 HTML，40/40 份匹配到课程页。
- 生成稳定性：连续两次生成哈希一致。

- 浏览器证据位于 `.playwright-cli/`（本地 QA 产物）。

## 发现与处理

### 已修复：中等 — Scala 代码被 Markdown 误解析为链接

**证据：** `new Array[Int](100)` 一度生成 `href="100"`，导致错误本地链接。

**影响：** 文章内容语义错误，链接扫描失败。

**处理：** 生成器现在保护 Scala 类型应用语法，且不会二次改写已在行内代码或围栏代码中的内容。增加两项回归测试。

### 已修复：中等 — 文章标题层级重复

**证据：** 4 篇 Markdown 含多个一级标题，与页面主标题重复。

**影响：** 屏幕阅读器的标题大纲不清晰，长文定位更困难。

**处理：** 生成时将 Markdown 标题统一下移一级；40 份文章现在均只有一个页面 `h1`。

### 已修复：中等 — 生成元数据的 HTML 注入边界

**证据：** 文章正文会转义原始 HTML，但文件名生成的标题和摘要最初直接写入 `innerHTML`。

**影响：** 当未来贡献者新增恶意文件名时，可能把 HTML 带入索引页。

**处理：** 标题、摘要、语言、日期和上/下标记在 DOM 模板中使用前统一转义，并新增注入回归测试。

### 已修复：低 — 全站 favicon 404

**证据：** 首次 Chromium 加载的唯一控制台错误为 `/favicon.ico` 404。

**处理：** 新增 `assets/icons/favicon.svg`，并在首页、课程页、总结索引、互动页与生成文章中显式引用。重新加载后无新控制台错误。

### 已修复：中等 — 移动端隐藏全部顶部导航

**证据：** 既有 `max-width: 900px` 规则使 `.nav-links` 完全不显示。

**影响：** 移动端用户难以在互动课程、课程总结和 Java 练习之间切换。

**处理：** 移动端导航改为第二行换行显示；375px 视口截图确认三个入口可见且无溢出。

### 剩余建议：中等 — 超长文章缺少页内目录

**证据：** 部分总结含十余个主要章节，移动端阅读距离很长。

**影响：** 回到某一知识点需要大量滚动。

**建议：** 后续可在生成器中为 `h2/h3` 生成稳定锚点和可折叠目录。这不影响本轮内容可用性，未在本次已批准范围中扩展。

### 剩余验证限制：低 — 未覆盖 Firefox/Safari 和真实辅助技术

**证据：** 本轮真实浏览器检查使用 Chromium；语义和键盘结构已检查，但未使用 NVDA/VoiceOver。

**建议：** 如果正式面向公众发布，增加 Firefox/WebKit 自动测试和 NVDA 手工测试。

- **Frontend Developer：** 纯静态架构与现有项目一致，无运行时框架负担；响应式卡片、代码和表格溢出处理合理。
- **UX Architect：** “总结资料馆 + 课程直达”将复习与互动学习清楚分层；长文目录是下一步最有价值的改进。
- **Accessibility Auditor：** 筛选标签、动态状态、标题层级、语义化文章与键盘链接结构通过检查；移动导航问题已处理。
