## 4. 知识点详细整理

### 知识点 1：类的基本概念

#### 这个知识点是什么？

**类（class）** 是 Java 面向对象编程的核心。类用来描述**一类事物共有的属性（property）和功能（method）**。

- **属性**（成员变量）：描述这个类"有什么"（如 name、age）
- **方法**（成员方法）：描述这个类"能做什么"（如 speak、study）

类就像一个**模板（template）**，对象是根据这个模板创建出来的**具体实例（instance）**。

#### 为什么要学它？

Java 的一切代码都写在类里面。不理解类的结构，就无法编写任何有意义的 Java 程序。类是 Java 程序的基本组织单位。

#### 基本语法

```java
public class 类名 {
    // 属性（property）：在上面
    数据类型 属性名;

    // 方法（method）：在下面
    public void 方法名() {
        // 方法体
    }
}
```

#### 课堂中的理解方式

老师用一个非常形象的类比——"大数据 24 级就是一个类，而每一位同学就是一个对象。类 = 模板（template），对象 = 实例（instance）。类是对对象的抽象，对象是类的实例。"

老师还特别强调了两条规则：
1. **类名遵循标识符规则**——不能使用汉语、不能以数字开头、不能是关键字。
2. **类体结构固定**——"属性在上面，方法在下面。这是一个约定俗成的规则，虽然不按这个写也没错，但看着不舒服。我要求大家按这个规则去写。"

#### 初学者容易犯的错误

```text
1. 类名用了中文——Java 标识符不能包含中文（虽然编译可能不报错，但极不推荐）。
2. 属性和方法的位置写反——方法写在了属性上面（虽然语法不报错，但不规范）。
3. 把 main 方法写在类外面——所有代码必须在类体 { } 内部。
```

### 知识点 2：成员变量（全局变量）与局部变量

#### 这个知识点是什么？

Java 中的变量按照**声明位置和作用范围**分为两大类：

| 类型 | 声明位置 | 可见范围 | 默认值 |
|------|---------|---------|--------|
| **成员变量**（全局变量） | 类体中、方法外 | 整个类体都可见 | 系统自动赋默认值（0/null/false） |
| **局部变量** | 方法体或代码块内 | 从声明处到方法结束 | **无默认值，必须手动初始化** |

**同名规则**：当局部变量与成员变量同名时，局部变量会**隐藏**（hide）成员变量。要访问被隐藏的成员变量，必须用 `this.变量名`。

#### 为什么要学它？

理解变量的作用范围是编程的基础。搞不清局部和全局，会出现"明明赋值了却读不到"或者"值莫名其妙被改了"的问题。

#### 课堂中的理解方式

老师用两个生动的比喻来解释：

**比喻一：全局变量 vs 局部变量的可见性**
- 成员变量 = "整个班的人都能看见他"——在任何位置都能访问。
- 局部变量 = "只在某个房间（方法）里能看见他"——出了这个房间就找不到了。
- "可见即可访问"——你只有能看到它，你才能使用它。

**比喻二：同名时的隐藏**
- "在一个文件里面，如果两个变量同名，局部变量就将全局变量隐藏。"
- "虽然两个 name 名字一样，但是是两个不同的 name——就像扬州有两个张三，一个叫大张三，一个叫小张三。"
- 要访问全局的那个 name，必须用 `this.name`。this 就是"当前对象"的引用。

#### 初学者容易犯的错误

```text
1. 局部变量忘记初始化就直接使用——编译错误！局部变量不像成员变量有默认值。
2. 两个同名变量搞混——在方法里直接写 name 访问的是局部变量，不是成员变量。
3. 在方法外部访问局部变量——"出了函数就看不到了"。
```

#### 正确示例

```java
public class Person {
    // ====== 成员变量（全局变量） ======
    String name;      // 默认值 null
    int age;          // 默认值 0

    public void speak(String name) {   // 这个 name 是局部变量（形参）
        // 局部变量 name 隐藏了成员变量 name
        System.out.println(name + "在说话");          // 访问的是局部变量 name
        System.out.println(this.name + "在说话");     // 访问的是成员变量 name（用 this）
    }

    public void test() {
        int x = 10;   // 局部变量——必须初始化！
        // ↑ 出了 test() 方法，x 就不可见了
        System.out.println(x);
    }
}
```

#### 示例解释

```text
1. name 和 age 是成员变量，在 Person 类中的任何方法里都能直接访问。
2. speak 方法的参数 name 是局部变量——它和成员变量 name 同名，所以在 speak 方法里直接写 name 访问的是参数（局部变量）。
3. 要访问被隐藏的成员变量 name，必须使用 this.name。
4. test() 中的 x 是局部变量，出了 test() 方法就不可见了。
5. 成员变量有默认值（name=null, age=0），但局部变量没有——x 必须手动初始化。
```

### 知识点 3：访问器——set 方法与 get 方法

#### 这个知识点是什么？

**访问器（Accessor）** 是一组成对的方法——**set 方法**（设置器/赋值）和 **get 方法**（获取器/取值）。它们的作用是**间接地**访问类的属性，而不是直接暴露属性给外部。

#### 为什么要学它？

直接暴露属性（如 `public String name`）是不安全的——任何代码都可以随意修改它。通过访问器可以：
1. **控制访问**——只给某些代码赋值的权限。
2. **验证数据**——比如年龄不能设为负数。
3. **提高程序的健壮性、安全性和稳定性**。

#### 基本语法

```java
// set 方法：给属性赋值
public void set属性名(数据类型 参数名) {
    this.属性名 = 参数名;
}

// get 方法：返回属性值
public 数据类型 get属性名() {
    return 属性名;
}
```

**命名规范**：set/get + 属性名（首字母大写），如 `setName` / `getName`。

#### 课堂中的理解方式

老师用**银行账户**做了精彩的类比：
- 账户余额就是一个属性——如果可以直接访问（`account.balance = 0`），那"钱还没花呢就没有了"，极不安全。
- 通过 set/get 方法（访问器）来控制——"通过函数来控制它。你给某一个人权限来访问，只有你能访问你的账户信息，别人访问不了。"
- "一般情况下，我们的属性是不直接赋值的。通过方法间接操作，提高了安全性。"

#### 初学者容易犯的错误

```text
1. 只写 set 方法忘写 get 方法——只能赋值不能读取，或者反过来。
2. set 方法中忘记写 this——导致自己赋值给自己（局部变量 = 局部变量），成员变量没被修改。
3. get 方法忘记写 return——这是取值的方法，必须有返回值。
```

#### 正确示例

```java
public class BankAccount {
    // 属性（private——不直接暴露）
    private String owner;
    private double balance;

    // ====== 访问器 ======
    // set 方法：设置账户名
    public void setOwner(String owner) {
        this.owner = owner;    // this.owner = 成员变量, owner = 参数
    }

    // get 方法：获取账户名
    public String getOwner() {
        return owner;
    }

    // set 方法：存钱（带验证）
    public void setBalance(double balance) {
        if (balance >= 0) {
            this.balance = balance;
        } else {
            System.out.println("余额不能为负数！");
        }
    }

    // get 方法：查余额
    public double getBalance() {
        return balance;
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount();
        acc.setOwner("张三");         // 通过 set 方法赋值
        acc.setBalance(1000.0);
        System.out.println(acc.getOwner() + " 余额：" + acc.getBalance());
    }
}
```

#### 示例解释

```text
1. owner 和 balance 都声明为 private——外部不能直接访问。
2. setOwner(String owner) 中，参数 owner 和成员变量 owner 同名，用 this.owner 来区分——this.owner 是成员变量，等号右边的 owner 是参数。
3. getOwner() 返回成员变量 owner 的值——这是"读取"操作。
4. setBalance 中加入了验证——如果余额设为负数就拒绝。这就是"通过方法控制属性"的直接体现。
```

### 知识点 4：this 关键字

#### 这个知识点是什么？

`this` 是一个关键字，代表**当前对象（current object）**的引用。谁在调用方法，this 就指向谁。

#### 为什么要学它？

当局部变量（方法参数）和成员变量同名时，必须用 `this` 来区分。`this.name` 表示"当前对象的 name 属性"，单独的 `name` 表示局部变量。

#### 课堂中的理解方式

老师用了一个非常接地气的比喻：
- "当前对象是谁？比如我说'班长，你去干什么'，那班长就是当前对象。"
- "我说'学委，你去干什么'，那学委就是当前对象。"
- **this 就是"不管谁是当前对象，我都用 this 来代替它"**。
- "谁在当前的、谁在做什么事情，就叫当前的。用 this 来引用它。"

#### 初学者容易犯的错误

```text
1. 忘记用 this——导致 set 方法中自己赋值给自己，成员变量没变。
2. 误以为 this 是一个对象——this 是引用，不是对象本身。
3. 在静态方法（static）中使用 this——静态方法没有"当前对象"的概念，不能使用 this。
```

### 知识点 5：参数传递——实参与形参

#### 这个知识点是什么？

方法定义时括号里写的参数叫**形式参数（形参）**——它只是一个"占位符"，没有实际值，等待被赋值。方法调用时传入的具体值叫**实际参数（实参）**——如 "Tom"、20、3.14 等具体数据。

将实参的值传给形参的过程叫做**参数传递（pass by value / pass by reference）**。

#### 课堂中的理解方式

老师解释："形参是在这等着、等着赋值，谁赋给我我就用谁。实参是实际存在的值——Tom、Peter、Alice、1、1.1、1.11。把实参传给形参，叫做参数传递。"

## 5. 重点概念对比

| 概念 A | 概念 B | 区别 | 示例 |
|---|---|---|---|
| 类（class） | 对象（object） | 类是模板/抽象；对象是具体实例 | `Student` 是类；`new Student()` 创建对象 |
| 属性（property） | 方法（method） | 属性是"有什么"（名词）；方法是"能做什么"（动词） | `String name;` / `void speak()` |
| 成员变量（全局） | 局部变量 | 全局在类体中处处可见，有默认值；局部只在方法内可见，必须初始化 | 类体中声明 vs 方法内声明 |
| 形参 | 实参 | 形参是方法定义时的占位符；实参是调用时传的具体值 | `void setName(String name)` 中 name 是形参 / `p.setName("Tom")` 中 "Tom" 是实参 |
| set 方法 | get 方法 | set 是赋值（写入）；get 是取值（读取） | `setName("Tom")` / `getName()` |
| `this.name` | `name` | this.name 访问成员变量；name 访问局部变量（同名时） | 在 set 方法中必须用 this |

## 6. 代码示例整理

### 示例 1：完整的类定义——从属性到访问器

**适用知识点**：

```text
类定义、属性、方法、构造方法、访问器
```

**代码**：

```java
public class Student {
    // 第1部分：属性
    private String name;
    private int age;

    // 第2部分：构造方法
    public Student() {
        name = "未命名";
        age = 0;
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 第3部分：访问器（get/set）
    public void setName(String name) { this.name = name; }
    public String getName() { return name; }
    public void setAge(int age) { this.age = age; }
    public int getAge() { return age; }

    // 第4部分：普通方法
    public void introduce() {
        System.out.println("我叫" + name + "，今年" + age + "岁。");
    }
}

// 测试类
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();              // 用无参构造
        s1.setName("小明");
        s1.setAge(18);
        s1.introduce();                          // 我叫小明，今年18岁。

        Student s2 = new Student("小红", 20);    // 用带参构造
        s2.introduce();                          // 我叫小红，今年20岁。
    }
}
```

### 示例 2：成员变量与局部变量同名时的区分

**适用知识点**：

```text
成员变量、局部变量、this 关键字、隐藏
```

**代码**：

```java
public class Person {
    String name = "张三";    // 成员变量

    public void test(String name) {   // 局部变量（参数）——与成员变量同名
        System.out.println("name = " + name);           // 访问局部变量
        System.out.println("this.name = " + this.name); // 访问成员变量
    }

    public static void main(String[] args) {
        Person p = new Person();
        p.test("李四");
    }
}
```

**运行结果**：

```text
name = 李四
this.name = 张三
```

## 7. 课堂案例：银行账户——理解访问器

**案例名称**：为什么银行卡余额不能被直接修改？

**案例背景**：老师用银行账户余额来帮助理解访问器——"你的银行账户，那个钱是属性。如果直接能赋值和取值，那钱就被划没了。"

**解决思路**：把属性设为 private（私有），外部代码不能直接访问。提供 set/get 访问器来间接操作——set 方法可以加验证逻辑（如"不能设为负数"），get 方法只返回余额。

**这个案例帮助理解什么**：理解"封装"的核心思想——把数据保护起来，通过受控的方法访问。

## 8. 易错点总结

```text
易错点 1：局部变量忘记初始化
错误示例：
public void test() {
    int x;
    System.out.println(x);   // 编译错误！
}

正确写法：
public void test() {
    int x = 0;
    System.out.println(x);   // 输出 0
}

解释：成员变量有默认值，但局部变量没有。局部变量使用前必须手动初始化。


易错点 2：set 方法中忘记 this，自己赋值给自己
错误示例：
public void setName(String name) {
    name = name;    // 两个 name 都是参数！成员变量没变！
}

正确写法：
public void setName(String name) {
    this.name = name;   // this.name = 成员变量, name = 参数
}

解释：不加 this 的话，等号左右两边都是局部变量（参数），成员变量纹丝不动。


易错点 3：get 方法忘记 return
错误示例：
public String getName() {
    name;    // 编译错误！没有 return
}

正确写法：
public String getName() {
    return name;
}

解释：get 方法必须有 return 语句返回要取的值。


易错点 4：void 方法里写 return 带值
错误示例：
public void test() {
    return 10;   // 编译错误！void 方法不能返回值
}

正确写法：
public void test() {
    // 不返回，或者只写 return;（不带值）
}

解释：void 表示"无返回值"，return 后面不能跟任何值。
```

## 9. 小测验题目

```text
题目 1：关于成员变量和局部变量，以下说法正确的是？

A. 成员变量和局部变量都有默认值
B. 局部变量有默认值，成员变量没有
C. 成员变量有默认值（如0、null），局部变量必须手动初始化
D. 两者都没有默认值

正确答案：C


题目 2：以下代码中，setName 方法执行后，成员变量 name 的值是什么？

public void setName(String name) {
    name = name;
}

A. 等于传入的参数值
B. 还是原来的值（没变）
C. null
D. 编译错误

正确答案：B

解析：两个 name 都是参数（局部变量）。参数赋值给参数，成员变量没有任何变化。应写 this.name = name;


题目 3：this 关键字的作用是什么？

A. 表示当前类的名称
B. 表示当前对象的引用
C. 表示父类的引用
D. 表示方法名
```

## 10. 拖拽匹配素材

```text
1. class —— Java 关键字，用来声明一个类
2. 属性（property） —— 类的成员变量，描述类"有什么"
3. 方法（method） —— 类的成员函数，描述类"能做什么"
4. 成员变量 —— 在类体中、方法外声明的变量，整个类体可见
5. 局部变量 —— 在方法体内声明的变量，只在方法内可见
6. this —— 代表当前对象的引用
7. set 方法 —— 访问器中的赋值方法，用于给属性设值
8. get 方法 —— 访问器中的取值方法，用于获取属性值
9. 形参 —— 方法定义时声明的参数，等待实参赋值
10. 实参 —— 方法调用时传入的具体值
```

## 11. 课后总结

```text
1. 本节课的核心是理解类的标准结构——属性在上、方法在下，方法是构造方法→访问器（get/set）→普通方法的顺序。
2. 学习面向对象时要特别注意成员变量和局部变量的区别——全局有默认值，局部必须初始化；同名时局部隐藏全局，需用 this 区分。
3. 访问器（get/set）是封装的核心机制——不要直接暴露属性，通过方法来安全地访问。
4. 这些概念会在后续的继承、多态、封装中反复使用——现在打好基础，后面才不会乱。
```
