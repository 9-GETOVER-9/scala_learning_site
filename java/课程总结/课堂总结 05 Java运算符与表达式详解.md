## 4. 知识点详细整理

### 知识点 1：运算符的分类

#### 这个知识点是什么？

Java 中的运算符（Operator）主要分为四大类：

| 类别 | 作用 | 常见运算符 |
|------|------|-----------|
| **算术运算符** | 进行数学计算 | `+` `-` `*` `/` `%` `++` `--` |
| **关系运算符** | 比较两个值的大小 | `>` `<` `>=` `<=` `==` `!=` |
| **逻辑运算符** | 进行逻辑判断 | `&&` `||` `!` |
| **位运算符** | 对二进制位进行操作 | `&` `|` `^` `~` `<<` `>>` |

#### 为什么要学它？

运算符是编写任何 Java 程序的基础。就像写作文需要用到标点符号和连接词一样，写代码需要用到运算符来表达计算和逻辑。没有运算符，程序将无法进行任何计算。

#### 课堂中的理解方式

老师指出课堂的重点是**算术运算符**和**逻辑运算符**，关系运算符相对简单（"大于小于是我们从小就学的"），位运算符本学期不要求深入掌握。

#### 初学者容易犯的错误

```text
1. 混淆赋值运算符 = 和比较运算符 ==（后面课程会重点讲）。
2. 混淆逻辑与 && 和位与 &（功能不同，用法不同）。
3. 不知道什么场景该用什么运算符。
```

### 知识点 2：运算级别与算术表达式

#### 这个知识点是什么？

Java 中的运算符有不同的**优先级**（运算级别）。优先级高的运算符先执行，优先级低的后执行。这和我们学数学时的"先乘除，后加减，有括号先算括号"规则完全一致。

- **3 级**（较高）：`*` `/` `%`（乘、除、求余）
- **4 级**（较低）：`+` `-`（加、减）
- **最高**：`()` 括号（有括号先算括号里的）

用算术符号和括号连接起来的式子叫做**算术表达式**。

#### 为什么要学它？

如果不懂优先级，写出的表达式可能得到意想不到的结果。比如 `1 + 2 * 3` 结果是 7 而不是 9——因为 `*` 优先于 `+`。

#### 基本语法

```java
int result = 1 + 2 * 3;          // result = 7（不是 9）
int result2 = (1 + 2) * 3;       // result2 = 9（括号改变了优先级）
int result3 = 10 / 3;            // result3 = 3（整数除法，小数被截断）
int result4 = 10 % 3;            // result4 = 1（求余：10 除以 3 的余数）
```

#### 课堂中的理解方式

老师这样解释：
- **运算符级别的含义**："就像数据类型有级别，运算也有级别。为什么要先算乘除后算加减？因为它有级别。"
- **计算机的执行顺序**："计算机运算是从上到下、从左到右，依次一行一行地读。遇到括号就先算括号里的值。"
- **人和计算机的区别**："人太聪明了，一眼就能看出等于多少，但计算机要一步一步按规则来。"

#### 初学者容易犯的错误

```text
1. 整数除法会自动截断，比如 5 / 2 结果是 2 而不是 2.5。
2. 求余运算 % 忘记了它的存在，或者不理解余数的含义。
3. 表达式写得太长太复杂，导致优先级混乱——建议用括号明确意图。
4. 混淆 / 和 %：/ 是商，% 是余数。
```

#### 正确示例

```java
public class OperatorPriority {
    public static void main(String[] args) {
        // 优先级示例
        int a = 10 + 2 * 3;       // 10 + 6 = 16
        int b = (10 + 2) * 3;     // 12 * 3 = 36
        System.out.println("a = " + a);   // 16
        System.out.println("b = " + b);   // 36

        // 求余运算
        int c = 10 / 3;           // 3（商）
        int d = 10 % 3;           // 1（余数）
        System.out.println("10 / 3 = " + c);   // 3
        System.out.println("10 % 3 = " + d);   // 1

        // 复杂表达式
        int k = 2 * (1 + 3) - 5 % 2;
        // 计算过程：2*4 - 1 = 8 - 1 = 7
        System.out.println("k = " + k);        // 7
    }
}
```

#### 示例解释

```text
1. 10 + 2 * 3：乘法的优先级高于加法，先算 2*3=6，再算 10+6=16。
2. (10 + 2) * 3：括号优先级最高，先算 10+2=12，再算 12*3=36。
3. 10 / 3 = 3：整数除法，只保留商，小数部分直接被丢弃。
4. 10 % 3 = 1：10 除以 3 得 3 余 1，% 取的是余数。
5. 2 * (1+3) - 5%2 = 2*4 - 1 = 8 - 1 = 7。
```

### 知识点 3：二目运算符与单目运算符

#### 这个知识点是什么？

根据运算符需要多少个**操作数**（参与计算的值），可以分为：

- **二目运算符**：需要**两个**操作数参与运算。如加减（`a + b`）、乘除（`a * b`）、求余（`a % b`）。
- **单目运算符**：只需要**一个**操作数参与运算。Java 中唯一的单目算术运算符就是自加（`++`）和自减（`--`）。

#### 为什么要学它？

理解二目和单目有助于理解自加自减的特殊性——它不跟别人算，只跟自己算（把自己加 1 或减 1）。

#### 基本语法

```java
// 二目运算符（需要两个操作数）
a + b;    // 加法
a - b;    // 减法
a * b;    // 乘法
a / b;    // 除法
a % b;    // 求余

// 单目运算符（只需要一个操作数）
a++;      // 自加（后置）
++a;      // 自加（前置）
a--;      // 自减（后置）
--a;      // 自减（前置）
```

#### 课堂中的理解方式

老师解释："加减叫二目运算，为什么叫二目？二目的意思就是有两个未知数在参与。A+B 或者 A-B，必须有两个数在参与。"

"自加自减是唯一的一个单目运算符——它只有一个操作员，就是 X 自己。作用就是自增或者自减。"

#### 初学者容易犯的错误

```text
1. 把自加自减当成二目运算符，写出 a++b 这种错误语法。
2. 以为 ++ 运算符可以用来加 2：a++++ 不是 a+2，而是 a 自加两次。
3. 对非数值类型使用自加自减——自加自减只能用于整数和浮点数。
```

#### 正确示例

```java
public class OperatorTypeDemo {
    public static void main(String[] args) {
        // 二目运算符
        int a = 10, b = 3;
        System.out.println("加法：" + (a + b));   // 13
        System.out.println("减法：" + (a - b));   // 7
        System.out.println("乘法：" + (a * b));   // 30
        System.out.println("除法：" + (a / b));   // 3
        System.out.println("求余：" + (a % b));   // 1

        // 单目运算符（自加自减）
        int x = 5;
        x++;    // 自加：x 变成 6
        System.out.println("自加后：" + x);       // 6

        int y = 10;
        y--;    // 自减：y 变成 9
        System.out.println("自减后：" + y);       // 9
    }
}
```

#### 示例解释

```text
1. 加、减、乘、除、求余都需要两个数参与——左边一个，右边一个。
2. 自加（x++）只需要 x 自己——把自己的值加 1，等价于 x = x + 1。
3. 自减（y--）只需要 y 自己——把自己的值减 1，等价于 y = y - 1。
```

### 知识点 4：自加（++）与自减（--）——前置与后置

#### 这个知识点是什么？

自加（`++`）和自减（`--`）是 Java 中**唯一的单目算术运算符**。它们可以让一个变量自增 1 或自减 1。关键在于运算符放在变量的**前面**还是**后面**，含义不同：

- **前置（++i 或 --i）**：**先自身运算，再参与表达式**。即变量先加 1/减 1，然后用新值参与其他运算。
- **后置（i++ 或 i--）**：**先参与表达式，再自身运算**。即先用变量的当前值参与其他运算，然后变量再加 1/减 1。

#### 为什么要学它？

自加自减是编程中最常用的运算符之一，尤其在循环中（如 `for(int i=0; i<10; i++)`）。不理解前置后置的差异，会导致计算结果完全错误，也是考试的重点考察内容。

#### 基本语法

```java
// 前置：先自身运算，再参与
int a = 1;
int b = ++a;    // a 先变成 2，然后把 2 赋给 b。结果：a=2, b=2

// 后置：先参与，再自身运算
int c = 1;
int d = c++;    // 先把 c 的当前值 1 赋给 d，然后 c 再变成 2。结果：c=2, d=1
```

#### 课堂中的理解方式

老师用了一个清晰的公式来讲解：

**前置加加（++i）的两个步骤：**
1. 第一步：`i = i + 1`（先把自己加 1）
2. 第二步：用加完后的 i（新值）参与表达式的其他运算

**后置加加（i++）的两个步骤：**
1. 第一步：先用 i 的当前值（原值）参与表达式的其他运算
2. 第二步：`i = i + 1`（运算完了再把自己加 1）

老师用了一个具体的例子："j = i++ 的话，j 等于什么？i 原先如果是 1，那 j = 1+2 = 3。如果 j = ++i，那就是 i 先变成 2，然后再 2+2 = 4。这两个值不一样。"

老师还提到英文术语：前置叫 **pre-increment**，后置叫 **post-increment**。pre- 是前缀，表示"在前面的"；post- 表示"在后面的"。

#### 初学者容易犯的错误

```text
1. 分不清前置和后置的区别——可以记口诀："前加先后，后加先后"（前置先加后用，后置先用后加）。
2. 在同一个表达式中多次对同一个变量使用 ++/--——如 int c = a++ + ++a; 结果因执行顺序而异，应避免。
3. 把自加自减和普通加法混淆——a++ 是 a 自己加 1，不是 a 加别的数。
4. 对常量使用自加：5++ 是错的——自加自减只能用于变量。
```

#### 正确示例

```java
public class IncrementDemo {
    public static void main(String[] args) {
        int a, b;

        // ====== 前置加加（先自身运算，再参与） ======
        int i = 1;
        int j = ++i + 2;     // 步骤1：i 先变成 2；步骤2：j = 2 + 2 = 4
        System.out.println("前置：i=" + i + ", j=" + j);  // i=2, j=4

        // ====== 后置加加（先参与，再自身运算） ======
        int m = 1;
        int n = m++ + 2;     // 步骤1：n = 1 + 2 = 3；步骤2：m 再变成 2
        System.out.println("后置：m=" + m + ", n=" + n);  // m=2, n=3

        // ====== 单独使用：前置和后置效果一样 ======
        int p = 5;
        p++;                 // p 变成 6（单独用时，p++ 和 ++p 效果一样）
        System.out.println("p = " + p);   // 6

        ++p;                 // p 变成 7
        System.out.println("p = " + p);   // 7

        // ====== 对比总结 ======
        int x = 3;
        int y = x++;   // y=3（先赋值）, x=4（后自加）
        int z = ++x;   // x=5（先自加）, z=5（再赋值）
        System.out.println("x=" + x + ", y=" + y + ", z=" + z);  // x=5, y=3, z=5
    }
}
```

#### 示例解释

```text
1. ++i + 2：i 先从 1 变成 2，然后 2 + 2 = 4 赋给 j。所以 i=2, j=4。
2. m++ + 2：m（当前值为 1）先参与运算，1 + 2 = 3 赋给 n，然后 m 再加 1 变成 2。所以 m=2, n=3。
3. 单独使用 p++ 或 ++p 效果完全一样——都是自加 1，因为没有"参与别的运算"这个步骤。
4. 最后的对比：x++ 后置（y 拿到了旧值 3），++x 前置（z 拿到了新值 5）。
```

### 知识点 5：空语句

#### 这个知识点是什么？

Java 中，以**分号**结尾的一段代码称为一条**语句**。而**空语句**就是一条只有分号、什么也不做的语句。它是 Java 程序中最简单的语句。

#### 为什么要学它？

理解空语句有助于理解"语句"和"分号"的关系。同时，有时多余的（或少写的）分号会导致空语句的产生，造成逻辑 bug——知道空语句的概念有助于排查这类问题。

#### 基本语法

```java
;    // 这就是一条空语句——什么也不做

// 典型场景：写循环时不小心多加了分号
for (int i = 0; i < 10; i++);   // 注意这个分号！它让 for 循环体变成了空语句
    System.out.println(i);        // 这一行不在循环里面！
```

#### 课堂中的理解方式

老师在课堂上提问"最简单的语句是什么？"，最终给出的答案是"空语句"——"我们说了，以分号结尾的称为语句。现在就一个分号，那就是一个语句，只不过什么也没有。"

#### 初学者容易犯的错误

```text
1. 在 if、for、while 的括号后面不小心多写了分号，导致后面的代码块不按预期执行。
2. 以为每个换行都需要一个分号——分号的作用是结束语句，不是结束行。
3. 忘记了分号，导致两条语句被当成一条——编译错误。
```

#### 正确示例

```java
public class EmptyStatementDemo {
    public static void main(String[] args) {
        ;;;;;;;    // 7 个空语句——什么都不做，但语法上合法

        int a = 10;   // 正常语句

        // ⚠️ 注意：if 后面多一个分号的陷阱
        int score = 55;
        if (score < 60);   // ← 这个分号让 if 的"执行体"变成了空语句！
            System.out.println("不及格");  // ⚠️ 这行不管条件怎样都会执行！

        // 正确写法（不要那个分号）：
        if (score < 60)
            System.out.println("真的不及格");
    }
}
```

#### 示例解释

```text
1. 连续的分号都是空语句——语法合法但毫无意义。
2. if (score < 60); 中的分号是 if 语句的"体"——一个空语句。所以 if 什么都不做。
3. 紧接着的 System.out.println("不及格") 已经不属于 if 了，它总是会执行。
4. 这是一个非常隐蔽的 bug，初学者很容易犯，也难以排查。
```

### 知识点 6：字符串拼接与类型自动转换

#### 这个知识点是什么？

在 Java 中，使用加号 `+` 可以连接字符串。如果 `+` 的一边是字符串，另一边是非字符串类型（如 int、double），Java 会自动将非字符串类型转换成字符串，然后拼接在一起。这叫做**字符串拼接**。

#### 为什么要学它？

在实际开发中，经常需要把文字和变量值一起输出（如调试信息、日志等）。字符串拼接是最常用的技巧之一。例如 `System.out.println("结果是：" + result)`。

#### 基本语法

```java
String str = "Hello" + "World";           // 两个字符串拼接
String str2 = "年龄：" + 18;               // 字符串 + int，int 自动转换为字符串
String str3 = "价格：" + 9.9;              // 字符串 + double
String str4 = "结果：" + (a + b);          // 先算括号里的加法，再拼接
```

#### 课堂中的理解方式

老师在输出 `"两个 K = " + k` 时解释："前面是一个字符串，后面是一个整型。它们加到一起，Java 会自动把整型变成字符串然后拼接起来。这就是字符串转换。"

老师还强调了一个初学者容易忽略的问题：**中英文符号的区别**。"中文括号和英文括号在输入方式中是不同的。中文括号不会被 Java 识别——一定要在英文输入法下写代码。分号也是一样，中文分号和英文分号长得不一样。"

#### 初学者容易犯的错误

```text
1. 拼接时忘记用括号把运算括起来：System.out.println("和：" + a + b) —— 输出的是 a 和 b 拼接（如 "1020"），而不是 a+b 的结果（30）。
2. 在拼接时使用了中文符号（中文括号、中文分号）导致编译错误。
3. 用中文输入法写代码——所有标点符号都必须用英文的。
```

#### 正确示例

```java
public class StringConcatDemo {
    public static void main(String[] args) {
        int a = 10, b = 20;

        // ✅ 正确：字符串 + 变量
        System.out.println("a 的值是：" + a);         // a 的值是：10

        // ✅ 正确：字符串 + 运算结果（用小括号括起来）
        System.out.println("a + b = " + (a + b));     // a + b = 30

        // ❌ 错误：忘记加括号
        System.out.println("a + b = " + a + b);       // a + b = 1020（不是我们想要的！）

        // 为什么？因为 Java 是从左到右依次拼接的：
        // "a + b = " + a → "a + b = 10"
        // "a + b = 10" + b → "a + b = 1020"

        // 多个变量拼接
        String name = "张三";
        int age = 20;
        double score = 88.5;

        // 输出：张三今年20岁，成绩是88.5分。
    }
}
```

#### 示例解释

| 概念 A | 概念 B | 区别 | 示例 |
|---|---|---|---|
| 二目运算符 | 单目运算符 | 二目需要 2 个操作数；单目只需要 1 个 | 二目：`a + b`；单目：`a++` |
| 前置 ++i | 后置 i++ | 前置：先自加再参与；后置：先参与再自加 | `j = ++i + 2;`（i 先变） / `j = i++ + 2;`（i 后变） |

| 空语句 `;` | 普通语句 | 空语句只有分号，什么也不做 | `;` vs `int a = 10;` |
| 字符串拼接 | 数值加法 | `+` 一边是字符串时是拼接；两边都是数值时是加法 | `"结果" + 5` → "结果5"；`2 + 3` → 5 |

## 6. 代码示例整理

### 示例 1：运算符优先级综合演练

**适用知识点**：

```text
运算级别、算术表达式、求余
```

**代码**：

```java
public class PriorityDemo {
    public static void main(String[] args) {
        // 注意：下面的表达式展示了优先级规则
        int r1 = 10 + 3 * 2;           // 10 + 6 = 16
        int r2 = (10 + 3) * 2;         // 13 * 2 = 26
        int r3 = 20 - 8 / 2;           // 20 - 4 = 16
        int r4 = 15 % 4 + 2;           // 3 + 2 = 5
        int r5 = 10 + 5 % 3;           // 10 + 2 = 12

        System.out.println("10 + 3 * 2 = " + r1);       // 16
        System.out.println("(10 + 3) * 2 = " + r2);     // 26
        System.out.println("20 - 8 / 2 = " + r3);       // 16
        System.out.println("15 % 4 + 2 = " + r4);       // 5
        System.out.println("10 + 5 % 3 = " + r5);       // 12

        // 演示括号的力量：括号可以改变默认优先级
        int withoutBracket = 10 + 20 * 3;    // 10 + 60 = 70
        int withBracket = (10 + 20) * 3;     // 30 * 3 = 90
        System.out.println("\n无括号：" + withoutBracket);  // 70
        System.out.println("有括号：" + withBracket);       // 90
    }
}
```

**运行结果**：

```text
10 + 3 * 2 = 16
(10 + 3) * 2 = 26
20 - 8 / 2 = 16
15 % 4 + 2 = 5
10 + 5 % 3 = 12

无括号：70
有括号：90
```

**代码解释**：

```text
1. 10 + 3 * 2：* 比 + 优先，先算 3*2=6，再算 10+6=16。
2. (10 + 3) * 2：括号改变优先级，先算 10+3=13，再算 13*2=26。
3. 15 % 4 = 3（15 除以 4 余 3），再加 2 得 5。
4. 有括号和无括号结果可以相差很大——不确定优先级时，加括号是最安全的做法。
```

**初学者提示**：

```text
- 口诀：乘除求余在前，加减在后，括号最大。
- 记不住优先级？用括号明确表达你的意图，代码也更易读。
```

### 示例 2：前置与后置自加自减完整对比

**适用知识点**：

```text
自加（++）、自减（--）、前置、后置
```

**代码**：

```java
public class PrePostDemo {
    public static void main(String[] args) {
        System.out.println("===== 前置 ++i =====");
        int i1 = 1;
        int j1 = ++i1 + 10;    // i1 先变成 2，然后 j1 = 2 + 10 = 12
        System.out.println("i1 = " + i1 + ", j1 = " + j1);  // i1=2, j1=12

        System.out.println("\n===== 后置 i++ =====");
        int i2 = 1;
        int j2 = i2++ + 10;    // j2 = 1 + 10 = 11（先用旧值），然后 i2 变成 2
        System.out.println("i2 = " + i2 + ", j2 = " + j2);  // i2=2, j2=11

        System.out.println("\n===== 单独使用（效果一样） =====");
        int a = 5;
        a++;                   // a = 6
        System.out.println("a++ 后：" + a);
        ++a;                   // a = 7
        System.out.println("++a 后：" + a);

        System.out.println("\n===== 自减对比 =====");
        int b1 = 5;
        int c1 = --b1;         // b1 先减为 4，然后赋给 c1
        System.out.println("--b1: b1=" + b1 + ", c1=" + c1);  // b1=4, c1=4

        int b2 = 5;
        int c2 = b2--;         // 先把 5 赋给 c2，然后 b2 减为 4
        System.out.println("b2--: b2=" + b2 + ", c2=" + c2);  // b2=4, c2=5
    }
}
```

**运行结果**：

```text
===== 前置 ++i =====
i1 = 2, j1 = 12

===== 后置 i++ =====
i2 = 2, j2 = 11

===== 单独使用（效果一样） =====
a++ 后：6
++a 后：7

===== 自减对比 =====
--b1: b1=4, c1=4
b2--: b2=4, c2=5
```

**代码解释**：

```text
1. ++i1 + 10：i1 先从 1 变成 2，再用 2+10=12。所以 i1=2, j1=12。
2. i2++ + 10：i2（值为 1）先参与 1+10=11，赋值给 j2，然后 i2 再变成 2。所以 i2=2, j2=11。
3. 同样是 +2 的操作，前置和后置得到的 j 值不同（12 vs 11），但 i 最后都是 2。
4. --b1 和 b2-- 的道理和 ++ 完全一样，只是把加 1 换成减 1。
5. 单独使用时（不参与表达式），前置和后置效果完全相同。
```

**初学者提示**：

```text
- 口诀：前加后用（前置：先加后用），先用后加（后置：先用后加）。
- 如果只是想让变量 +1，用 i++ 就行（最简洁）。
- 在复杂的表达式中避免混用前置和后置——拆开来写更清晰。
```

### 示例 3：空语句陷阱演示

**适用知识点**：

```text
空语句、if 陷阱、分号
```

**代码**：

```java
public class EmptyStatementTrap {
    public static void main(String[] args) {
        int score = 55;

        // ❌ 陷阱：if 后面多了一个分号
        System.out.println("===== 有陷阱的版本 =====");
        if (score >= 60);    // ← 这个分号让 if 变得"什么都不做"
            System.out.println("及格了！");   // 这句话不管条件怎样都会执行！

        // ✅ 正确版本
        System.out.println("\n===== 正确的版本 =====");
        if (score >= 60)
            System.out.println("及格了！");
        else
            System.out.println("不及格，继续努力！");

        // 说明
        System.out.println("\n===== 解释 =====");
        System.out.println("上面先打印了'及格了！'，是因为 if 的分号导致了空语句陷阱。");
        System.out.println("如果不相信，可以试试把 score 改成 30 再运行。");
    }
}
```

**运行结果**：

```text
===== 有陷阱的版本 =====
及格了！

===== 正确的版本 =====
不及格，继续努力！

===== 解释 =====
上面先打印了'及格了！'，是因为 if 的分号导致了空语句陷阱。
```

**代码解释**：

```text
1. if (score >= 60); —— 注意这里的分号！它让 if 条件后面跟了一个空语句。
2. 所以无论 score 是 55 还是 95，if 都是"如果条件成立，什么都不做（空语句）"。
3. 紧接着的 System.out.println("及格了！"); 不属于 if，它总是会执行。
4. 正确写法是去掉 if 后面的分号，让 println 语句成为 if 的执行体。
```

**初学者提示**：

```text
- if、for、while 的括号后面**不要随便加分号**！
- 如果不小心多写了分号，编译器通常不会报错（语法上合法），但逻辑上完全是错的。
- 这是初学者最隐蔽的 bug 之一，排查起来很费时间。
```

## 7. 课堂案例整理

### 案例 1：计算器原理——运算符优先级

- **案例名称**：为什么计算器知道先乘除后加减？
- **对应知识点**：运算优先级
- **案例背景**：老师问"为什么我们手机上的计算器输入一串式子，它就能给出正确结果？"
- **解决思路**：因为程序员把运算规则（优先级）写到了计算器的代码里。当你输入 `1+2*3`，计算器知道 `*` 优先级更高，所以先算 `2*3=6`，再算 `1+6=7`。人输入的值只是"参数"，规则是程序提前定好的。
- **这个案例帮助理解什么**：理解"程序 = 规则 + 数据"的基本思想。优先级不是计算机自动知道的，而是被人为定义的。

### 案例 2：自加自减的两种角色

- **案例名称**：排队——先排队再做事，还是先做事再排队？
- **对应知识点**：前置 ++ 与后置 ++
- **案例背景**：用日常场景帮助理解前置和后置的区别。
- **解决思路**：
  - **前置（++i）**：先排队（先自加 1），再做事（参与表达式计算）。
  - **后置（i++）**：先做事（用当前值参与运算），再排队（运算完再加 1）。
  - 如果只是单独一个人站着（单独使用），先排队和后排队没区别——反正排完就算了（i 的值都加了 1）。

## 8. 易错点总结

```text
易错点 1：整数除法的结果也是整数（小数被截断）
错误示例：
int result = 5 / 2;    // 以为等于 2.5

实际情况：
result = 2（小数部分被截断）

解释：
整数除以整数，结果还是整数。要得到小数结果，需要把操作数之一转为 double：5.0 / 2 = 2.5。


易错点 2：混淆前置 ++i 和后置 i++ 的效果
错误示例：
int i = 1;
int j = i++;    // 误以为 j = 2

实际情况：
j = 1, i = 2（后置：先用旧值 1 赋值，i 再加 1）

解释：
后置 ++ 是"先用后加"。想先加后用，要写 ++i（前置）。


易错点 3：if 后面多写分号
错误示例：
if (score >= 60);
    System.out.println("及格");

实际情况：
分号让 if 后面跟了一个空语句，"及格"总是会被输出，不管 score 是多少。

解释：
if、for、while 的括号后面不要随便加分号。这是非常隐蔽的 bug。


易错点 4：字符串拼接时忘记加括号
错误示例：
System.out.println("和：" + a + b);   // a=1, b=2 → 输出 "和：12"

正确写法：
System.out.println("和：" + (a + b)); // 输出 "和：3"

解释：
Java 从左到右依次处理 + 号。第一个 + 是字符串拼接，后面的 + 也都变成了拼接。加括号可以改变运算顺序。


易错点 5：使用中文标点符号
错误示例：
System.out.println（"你好"）；   // 中文括号和中文分号

正确写法：
System.out.println("你好");      // 全部使用英文标点

解释：
中文括号（）和英文括号()在 Java 中不是同一个字符！Java 只认英文符号。写代码时一定要切换到英文输入法。
```

## 11. 流程动画素材

```text
前置 ++i 与后置 i++ 的执行流程对比：

【前置 ++i（以 j = ++i + 2 为例）】
1. JVM 看到 ++i，这是前置自加。
2. 第一步：i 先自加 1（i 变成新值）。
3. 第二步：用 i 的新值参与后面的运算（j = i + 2）。
4. 得到最终结果。

【后置 i++（以 j = i++ + 2 为例）】
1. JVM 看到 i++，这是后置自加。
2. 第一步：先用 i 的当前值参与后面的运算（j = i + 2）。
3. 第二步：等整个表达式算完后，i 再自加 1。
4. 得到最终结果。

【关键区别】
- 前置：i 先变，再参加别人的计算。
- 后置：i 先参加别人的计算，再自己变。
- 单独使用 i++ 或 ++i 时，没有"别人"，所以效果一样（i 就是自加 1）。
```

```text
public class Average {

}

解题思路：
整数除以整数结果是整数（截断小数）。要得到精确结果，需要让除数变成 double（除以 3.0 而不是 3），或者进行强制类型转换。
```

## 15. 课后总结

```text
1. 本节课的核心是掌握 Java 运算符的两大重点：运算优先级（乘除求余高于加减，括号最高）和自加自减的前置/后置区别。
2. 学习运算符时要特别注意前置 ++i（先加后用）和后置 i++（先用后加）的差异——这是每次考试必考的内容，也是初学阶段最容易出错的知识点。
3. 通过本节课，应该能够正确计算表达式、使用 ++ 和 -- 控制变量、用 % 求余数、用 + 拼接字符串输出结果。
4. 自加自减（++、--）在后续学习的 for 循环中会大量出现，它是循环控制的基础语法。现在掌握了，后面学循环就会轻松很多。
5. 课后建议多练习：(1) 手算几个含不同优先级的表达式；(2) 对比前置和后置自加的结果并解释原因；(3) 练习用 % 判断奇偶数（n % 2 == 0 就是偶数）；(4) 小心 if 后面的分号陷阱，写出正确和错误的版本对比运行结果。
```
