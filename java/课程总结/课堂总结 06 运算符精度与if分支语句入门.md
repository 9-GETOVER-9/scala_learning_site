## 4. 知识点详细整理

### 知识点 1：算术运算符的精度规则

#### 这个知识点是什么？

当表达式中出现**不同类型**的数据时，Java 会自动进行**向上转换**——以表达式中**精度最高**的类型为基准进行计算。具体规则是：

- 如果表达式中有 **double**（双精度），整个表达式按 **double** 运算，结果也是 double。
- 如果表达式最高级是 **float**（单精度），按 **float** 运算。
- 如果全是 **int**（整型），按 **int** 运算（注意：整数除法会截断小数）。
- **char** 类型在运算中会自动转换为 int 类型。

#### 为什么要学它？

如果不理解精度规则，就会困惑为什么 `5 / 2` 结果是 2 而不是 2.5，为什么 `5.0 / 2` 结果是 2.5。这直接关系到计算结果的正确性。

#### 基本语法

```java
// 自动提升规则
int a = 5;
double b = 2.0;
double result = a / b;   // a 自动提升为 double，结果 = 2.5

// 全是 int 的情况
int c = 5, d = 2;
int result2 = c / d;     // 整数除法，结果 = 2（小数截断）

// char 参与运算
char ch = 'A';
int code = ch + 1;       // ch 自动转成 ASCII 码 65，然后 +1 = 66
```

#### 课堂中的理解方式

老师提出了三条核心规则：

1. **有 double 就按 double**："只要在这个式子里边有一个数是 double，那整个这个东西都是 double。因为要向上转换。不向上转换的话，它就加不了。"
2. **按最高精度来**："表达式最高级是单浮点就按单浮点做，是整型就按整型做。"
3. **运算时左右两边类型要一致**："在进行运算的时候，我们要保证运算数左右两边类型保持一致。不一样的话，要往上升或者往下降。"

老师还提到了 char 的转换："char 可以变成 int。"

#### 初学者容易犯的错误

```text
1. 两个 int 相除期待得到小数结果——int / int 结果还是 int，小数部分被截断。
2. 忘记用 3.0 而不是 3 来确保浮点除法——int 除以 double（3.0）结果才是 double。
3. 在混合运算中搞不清最终结果是什么类型——记住口诀：按表达式中最高精度来。
```

#### 正确示例

```java
public class PrecisionDemo {
    public static void main(String[] args) {
        // 全 int 运算 → 结果是 int
        int a = 10, b = 3;
        System.out.println("int / int：" + (a / b));        // 3

        // 有 double → 升级为 double
        System.out.println("有 double：" + (a / 3.0));      // 3.3333...

        // 表达式中有 double，整体按 double
        double result = a / 2.0 + 1;
        System.out.println("混合运算：" + result);           // 6.0

        // char 参与运算 → 自动转 int
        char ch = 'A';
        int code = ch + 1;
        System.out.println("'A' + 1 = " + code);            // 66（B 的 ASCII 码）

        // float 和 double 混合
        float f = 3.5f;
        double d = 2.0;
        double mixed = f + d;   // float + double → 整体按 double
        System.out.println("float + double：" + mixed);     // 5.5
    }
}
```

#### 示例解释

```text
1. 10 / 3 = 3：两个 int 相除，结果还是 int，小数部分（.3333...）被截断。
2. 10 / 3.0 = 3.3333...：因为 3.0 是 double，int 的 10 自动转成 double，按 double 除法计算。
3. char 'A' = 65（ASCII 码），'A' + 1 = 66 = 'B'。
4. float + double：float 自动提升为 double，结果也是 double。
```

### 知识点 2：逻辑非（!）运算符

#### 这个知识点是什么？

逻辑非 `!` 是一个单目逻辑运算符，用于**反转**布尔值。`!true` 变成 `false`，`!false` 变成 `true`。

#### 为什么要学它？

程序中有大量"如果不是……"的判断——比如"如果不是周末"、"如果密码不正确"。这些场景都需要用 `!` 来取反。

#### 基本语法

```java
!布尔表达式    // 取反

// 示例
boolean isRain = true;
if (!isRain) {             // !true = false，不执行这里
    System.out.println("不下雨，出门");
}
```

#### 课堂中的理解方式

老师用一句话解释了逻辑非："非就是如果是 true 就变成 false，如果是 false 就变成 true。"逻辑非、逻辑与（&&）、逻辑或（||）构成 Java 的三种逻辑运算符。

#### 初学者容易犯的错误

```text
1. 把 ! 写在变量后面——! 必须写在布尔表达式前面。
2. 对非布尔类型使用 !——! 只能用于 boolean 类型的表达式。
3. 双重否定 !!（!(!true) = true），虽然语法正确但容易让人困惑，应避免。
```

#### 正确示例

```java
public class LogicalNotDemo {
    public static void main(String[] args) {
        boolean isLoggedIn = false;

        // 用 ! 判断"未登录"
        if (!isLoggedIn) {          // !false = true，执行
            System.out.println("请先登录！");
        }

        // ! 可以直接用在比较结果上
        int score = 85;
        if (!(score < 60)) {        // !(false) = true
            System.out.println("及格了");
        }

        // 等价于更直观的写法
        if (score >= 60) {
            System.out.println("及格了");   // 结果相同，但这样写更易读
        }
    }
}
```

#### 示例解释

```text
1. !isLoggedIn：isLoggedIn 是 false，取反后是 true，所以执行"请先登录！"。
2. !(score < 60)：score=85，85<60 是 false，取反后是 true，执行"及格了"。
3. 虽然 !(score < 60) 和 score >= 60 结果一样，但后者更符合人的阅读习惯——能用正面的表达就不建议用双重否定。
```

### 知识点 3：if 语句——最简单的条件判断

#### 这个知识点是什么？

`if` 语句（条件判断语句）让程序能够根据条件是否成立来决定**是否执行**某段代码。这是 Java 中最基本的流程控制语句。

- 如果 if 后面括号里的条件为 **true**，就执行后面花括号 { } 里的代码。
- 如果条件为 **false**，就**跳过**花括号里的代码，继续执行后面的内容。

#### 为什么要学它？

现实世界中充满了选择——"如果下雨就带伞""如果及格就庆祝"。编程也是一样，程序需要根据不同的情况做出不同的响应。if 语句就是实现这种"根据条件做选择"的基本工具。

#### 基本语法

```java
// 格式一：不带花括号（只影响紧跟着的一句）
if (条件)
    一条语句;

// 格式二：带花括号（推荐，影响整个代码块）
if (条件) {
    语句1;
    语句2;
    ...
}
```

#### 课堂中的理解方式

老师用一个清晰的流程图来解释 if 的执行过程：
- "if 这个地方是一个条件表达式（condition），它要判断这个值是 true 还是 false。"
- "如果是 true，我们就执行里面的 body。如果是 false，我们就不执行它，然后跳出来。"
- "跳出它的 body，跳出语句组。"

老师还举了个例子：`if (x == 0)`，"这个表达式是真是假？如果是真，我们就执行下面这一段。"

#### 初学者容易犯的错误

```text
1. if 后面的括号里写了赋值语句而不是比较：if (x = 0) 是错误的，应该用 if (x == 0)。
2. if 后面多写分号：if (x > 0); 导致 if 什么都不判断（空语句陷阱）。
3. 忘记写花括号时，if 只影响紧跟着的一条语句，第二条语句无论条件如何都会执行。
4. 条件表达式忘记写括号：if x > 0 是语法错误，必须写 if (x > 0)。
```

#### 正确示例

```java
public class IfDemo {
    public static void main(String[] args) {
        int score = 85;

        // 基本 if 语句
        if (score >= 60) {
            System.out.println("恭喜，你及格了！");
            System.out.println("你的分数是：" + score);
        }

        // if 条件不成立的情况
        int temperature = 25;
        if (temperature > 30) {        // 25 > 30 是 false，跳过不执行
            System.out.println("天气炎热，开空调。");
        }
        System.out.println("程序继续运行...");

        // 对比：不写花括号时的陷阱
        int num = 5;
        if (num > 3)
            System.out.println("第一行");   // 受 if 控制
            System.out.println("第二行");   // ⚠️ 不受 if 控制！总是执行

        // 推荐：始终写花括号，哪怕只有一条语句
        if (num > 3) {
            System.out.println("第一行");   // 受 if 控制
        }
    }
}
```

#### 示例解释

```text
1. if (score >= 60)：85 >= 60 是 true，所以执行括号内的两行输出。
2. if (temperature > 30)：25 > 30 是 false，所以跳过，直接执行后面的"程序继续运行..."。
3. 不写花括号时，if 只控制紧接着的一句。"第二行"不管 num 是多少都会执行。
4. 推荐始终使用花括号——这是代码规范，也是避免 bug 的好习惯。
```

### 知识点 4：if-else 分支语句

#### 这个知识点是什么？

`if-else` 是 if 语句的升级版，提供了**两条路**：
- 条件为 **true** → 执行 **if 块**（statement one）。
- 条件为 **false** → 执行 **else 块**（statement two）。

两条路**必走其一**，不会同时走，也不会一条都不走。

#### 为什么要学它？

生活中很多场景只有两种可能——"及格 / 不及格""登录 / 未登录""成年 / 未成年"。if-else 就是用来处理这种二选一场景的。

#### 基本语法

```java
if (条件) {
    // 条件为 true 时执行
    语句1;
} else {
    // 条件为 false 时执行
    语句2;
}
```

#### 课堂中的理解方式

老师用了一个很直观的比喻：
- "如果怎么样，我们就这样。如果这个地方不怎么样，我们就那样。"
- "这叫**分支语句**——它有两个分支，两条路。这条路可能是这个，那条路可能是这个。"
- "有选择——如果嘛，不是选择嘛。你选择如果是真就走这边，如果是假就走那边。"

老师还画了一个流程图来说明：一个判断点（菱形），true 走一条路，false 走另一条路。

#### 初学者容易犯的错误

```text
1. else 后面直接跟条件——else 不带条件，else 后面直接写 { }。
2. else 和 if 之间写了分号：if (x>0); else { } 是错误，if 后面不能有分号。
3. 混淆 else if 和 else——else if 是"不满足第一个，再判断第二个条件"；else 是"以上都不满足就执行这个"。
```

#### 正确示例

```java
public class IfElseDemo {
    public static void main(String[] args) {
        int score = 75;

        // 基本 if-else
        if (score >= 60) {
            System.out.println("及格！");
            System.out.println("你的分数是：" + score);
        } else {
            System.out.println("不及格，继续努力！");
            System.out.println("你的分数是：" + score);
        }

        // 判断奇偶数
        int num = 7;
        if (num % 2 == 0) {
            System.out.println(num + " 是偶数");
        } else {
            System.out.println(num + " 是奇数");
        }

        // 判断成年与否
        int age = 16;
        if (age >= 18) {
            System.out.println("已成年，可以进入");
        } else {
            System.out.println("未成年，不能进入");
        }
    }
}
```

#### 示例解释

```text
1. score=75，75>=60 是 true，执行 if 块："及格！你的分数是：75"。
2. num=7，7%2=1，1==0 是 false，执行 else 块："7 是奇数"。
3. age=16，16>=18 是 false，执行 else 块："未成年，不能进入"。
4. 在 if-else 结构中，永远只执行其中一个分支——不会两个都执行，也不会一个都不执行。
```

### 知识点 5：代码规范——大小写与输入习惯

#### 这个知识点是什么？

Java 是一门**严格区分大小写**的语言。`System` 的 S 必须大写，`system` 或 `SYSTEM` 都是错的。这不仅关系到语法正确性，也关系到考试得分。

#### 为什么要学它？

因为大小写问题导致的编译错误是初学者最常见的错误之一。养成良好的大小写习惯，不仅能减少 bug，也让代码看起来更专业。

#### 课堂中的理解方式

老师非常严肃地强调了这个点：
- "这个 S 是大写，不要写成小写。考试的时候，有很多人说'老师我都写的不错呀，怎么能错了呢？'——如果我看你这个小写的 s，写成和 g 一边高的，那 sorry，你错了。"
- "你必须把这个 S 写成明显的大写。如果你写成 s y s 这样的，对不起，错。"
- "一定不要因为你的输入导致你自己的记忆力低下。"

换句话说，**输入习惯不好 = 分数低**。老师是在用考试来倒逼学生养成正确的输入习惯。

#### 初学者容易犯的错误

```text
1. System 的 S 写成小写 s（system）。
2. String 的 S 写成小写 s（string）。
3. println 写成 Println 或 printLine（正确全小写：println）。
4. main 写成 Main（正确全小写：main）。
5. 中文输入法下输入符号——中文分号、中文括号 Java 不识别。
```

#### 正确示例

```java
// ✅ 正确的大小写
public class CodeStyle {
    public static void main(String[] args) {   // String S大写, args 小写
        System.out.println("Hello");            // System S大写, out 小写, println 全小写
    }
}

// ❌ 错误的大小写示例（无法编译）
// public class codestyle {
//     public static void Main(string[] args) {    // Main 应小写, string 应大写String
//         system.out.println("Hello");             // system 应大写System
//     }
// }
```

## 5. 重点概念对比

| 概念 A | 概念 B | 区别 | 示例 |
|---|---|---|---|
| int / int | int / double | int/int 结果是 int（截断）；int/double 结果是 double（保留小数） | `5/2=2` / `5/2.0=2.5` |
| char 参与运算 | int 参与运算 | char 会自动转成 ASCII 码（int）再参与运算 | `'A'+1 = 66` |
| if | if-else | if 只有一条路（条件成立才执行）；if-else 有两条路（必走其一） | `if(score>=60) {...}` / `if(score>=60){...} else{...}` |
| 逻辑非 ! | 比较运算符 | ! 反转布尔值；==、!= 等比较两个值 | `!true = false` / `5 == 3` 结果 false |
| 运算精度：有 double | 运算精度：全 int | 有 double 时整个表达式按 double 算；全 int 时按 int 算（截断小数） | `10/3.0=3.33...` / `10/3=3` |
| System（大写S） | system（小写s） | System 是 Java 标准类名（必须大写）；小写 system 编译错误 | 这直接关系到代码是否能运行 |

## 6. 代码示例整理

### 示例 1：不同精度类型的混合运算

**适用知识点**：

```text
运算精度、自动类型提升、char 参与运算
```

**代码**：

```java
public class MixedPrecisionDemo {
    public static void main(String[] args) {
        // ====== 场景 1：全 int → 结果是 int ======
        int x = 7, y = 2;
        int r1 = x / y;              // 整数除法
        System.out.println("7 / 2（全int）= " + r1);       // 3

        // ====== 场景 2：有 double → 结果是 double ======
        double r2 = x / 2.0;         // 2.0 是 double，x 自动提升
        System.out.println("7 / 2.0（有double）= " + r2);  // 3.5

        // ====== 场景 3：混合类型 ======
        double a = 5.5;
        int b = 2;
        double r3 = a * b;           // b 自动提升为 double
        System.out.println("5.5 * 2 = " + r3);            // 11.0

        // ====== 场景 4：char 参与运算 ======
        char c1 = 'A';
        char c2 = 'B';
        int sum = c1 + c2;           // 'A'(65) + 'B'(66) = 131
        System.out.println("'A' + 'B' = " + sum);         // 131

        // ====== 场景 5：double 强制转 int ======
        double pi = 3.14;
        int intPi = (int) pi;        // 强制转换，截断小数
        System.out.println("(int)3.14 = " + intPi);       // 3
    }
}
```

**运行结果**：

```text
7 / 2（全int）= 3
7 / 2.0（有double）= 3.5
5.5 * 2 = 11.0
'A' + 'B' = 131
(int)3.14 = 3
```

**代码解释**：

```text
1. 7/2=3：两个 int 相除，小数被舍弃。
2. 7/2.0=3.5：2.0 是 double，7 自动转成 7.0，按浮点除法计算，保留小数。
3. 5.5*2=11.0：b=2 自动提升为 2.0，然后 5.5*2.0=11.0。
4. 'A'+'B'=65+66=131：char 在运算中自动转成对应的 ASCII/Unicode 码。
5. (int)3.14=3：强制转换截断小数（非四舍五入）。
```

**初学者提示**：

```text
- 想要除出小数结果 → 把其中一个操作数写成 3.0（double）而不是 3（int）。
- char 的 ASCII 码：'A'=65, 'B'=66, 'a'=97, '0'=48。
```

### 示例 2：if 和 if-else 的完整对比

**适用知识点**：

```text
if 语句、if-else 分支语句、条件判断
```

**代码**：

```java
public class IfVsIfElse {
    public static void main(String[] args) {
        int score = 55;

        // ====== 只使用 if（两个独立判断） ======
        System.out.println("===== 两个独立的 if =====");
        if (score >= 90) {
            System.out.println("优秀");
        }
        if (score >= 60) {
            System.out.println("及格");
        }
        if (score < 60) {
            System.out.println("不及格");
        }
        // 每次判断都是独立的，55 < 60，所以只输出"不及格"

        // ====== 使用 if-else if-else（互斥分支） ======
        System.out.println("\n===== if-else if-else =====");
        if (score >= 90) {
            System.out.println("优秀");
        } else if (score >= 60) {
            System.out.println("及格");
        } else {
            System.out.println("不及格");
        }
        // 三个分支互斥，只会走一个：55 不满足前两个，走 else

        // ====== 基本 if-else（二选一） ======
        System.out.println("\n===== 基本 if-else =====");
        int age = 20;
        if (age >= 18) {
            System.out.println("成年人");
        } else {
            System.out.println("未成年人");
        }
    }
}
```

**运行结果**：

```text
===== 两个独立的 if =====
不及格

===== if-else if-else =====
不及格

===== 基本 if-else =====
成年人
```

**代码解释**：

```text
1. 两个独立的 if：每个 if 独立判断，互不影响。score=55，前两个都是 false，第三个是 true。
2. if-else if-else：这是一个整体结构，三个分支互斥。只要前面的条件满足了，后面的就不再判断。
3. if-else 二选一：最简单的分支结构，不是走 if 就是走 else。
4. 什么时候用独立 if？——多个不互斥的条件（如多个独立的功能判断）。
5. 什么时候用 if-else？——互斥的条件（如成绩等级划分，一个分数只能对应一个等级）。
```

**初学者提示**：

```text
- if-else 中，else 后面不要再写条件（如 else (score < 60) 是错的）。
- 如果要用多个条件，写成 else if（连在一起的两个单词）。
```

### 示例 3：用 if-else 做简单计算器

**适用知识点**：

```text
if-else、逻辑判断、运算符
```

**代码**：

```java
public class SimpleCalculator {
    public static void main(String[] args) {
        int a = 15, b = 6;
        char operator = '/';    // 运算符：+、-、*、/

        System.out.println("计算：" + a + " " + operator + " " + b);

        if (operator == '+') {
            System.out.println("结果：" + (a + b));
        } else if (operator == '-') {
            System.out.println("结果：" + (a - b));
        } else if (operator == '*') {
            System.out.println("结果：" + (a * b));
        } else if (operator == '/') {
            if (b != 0) {
                System.out.println("结果：" + (a / b));         // 整数除法
                System.out.println("精确结果：" + (a * 1.0 / b)); // 浮点除法
            } else {
                System.out.println("错误：除数不能为零！");
            }
        } else {
            System.out.println("错误：不支持的运算符！");
        }
    }
}
```

**运行结果**：

```text
计算：15 / 6
结果：2
精确结果：2.5
```

**代码解释**：

```text
1. 用 if-else if-else 根据不同的运算符执行不同的计算。
2. operator == '/' 成立，进入除法分支。
3. 检查除数 b 是否为 0（防御性编程，避免除零错误），b=6 ≠ 0，继续。
4. a/b=15/6=2（整数除法）；a*1.0/b=15.0/6=2.5（浮点除法）。
5. 如果运算符不是 +、-、*、/ 中的任何一个，走最后的 else 输出错误提示。
```

**初学者提示**：

```text
- == 是比较运算符，判断是否相等，不同于 =（赋值）。
- 写计算器时记得检查除数是否为 0。
- a * 1.0 可以巧妙地将 int 转为 double，实现浮点除法。
```

```java
int x = 0;
int y = 10;
if (x == 0) {
    y = 20;
}
// x==0 是 true，所以 y 被赋值为 20
```

- **这个案例帮助理解什么**：帮助理解"if 只认布尔值（true/false），根据这个值决定走哪条路"。

## 8. 易错点总结

```text
易错点 1：全 int 除法期待得到小数
错误示例：
int result = 5 / 2;      // 期望 2.5，实际得到 2

正确写法：
double result = 5.0 / 2; // 或 5 / 2.0，结果才是 2.5

解释：
int/int 结果一定是 int（小数截断）。要得到 double 结果，至少有一个操作数是 double。


易错点 2：if 后面多写分号（空语句陷阱）
错误示例：
if (x > 0);
    System.out.println("正数");    // ⚠️ 总是输出！

正确写法：
if (x > 0)
    System.out.println("正数");    // 分号去掉

解释：
分号让 if 后面跟了一个空语句。println 不在 if 里面了，条件不再影响它。


易错点 3：把赋值 = 当成比较 ==
错误示例：
if (x = 0)     // 编译错误：int 不能当作 boolean

正确写法：
if (x == 0)    // 比较 x 是否等于 0

解释：
= 是赋值，== 是比较。if 的括号里必须是 boolean 类型表达式。这是初学者最容易犯、也是最难排查的错误之一。


易错点 4：else 后面写了条件
错误示例：
if (score >= 60) { ... }
else (score < 60) { ... }    // 错误：else 后面不能跟条件

正确写法：
if (score >= 60) { ... }
else { ... }                  // else 直接跟花括号

解释：
else 表示"以上所有条件都不满足"，不需要（也不能）再写条件。如果要加新条件，用 else if。


易错点 5：混淆 char 和 int 在运算中的行为
错误示例：
char c = 'A';
int n = c;          // n 是 65（正确，自动转换）
char c2 = c + 1;    // 编译错误！c+1 结果是 int，不能直接赋给 char

正确写法：
char c2 = (char)(c + 1);  // 需要强制转换成 char

解释：
char 参与运算时自动转为 int，运算结果是 int 类型。如果要把结果存回 char，需要强制转换 (char)。
```

## 9. 小测验题目

```text
题目 1：以下表达式中，哪个的结果是 double 类型？

A. 10 / 3
B. 10 / 3.0
C. 10 % 3
D. (int) 3.14

正确答案：B

解析：
10/3.0 中有一个 double（3.0），表达式整体按 double 计算。A 是全 int 结果为 int；C 求余也是 int；D 强制转 int 后也是 int。


题目 2：以下代码的输出结果是什么？

int a = 10;
int b = 3;
double c = a / b;
System.out.println(c);

A. 3.3333...
B. 3.0
C. 3
D. 编译错误

正确答案：B

解析：
a/b 是 int/int = 3（整数除法），然后把 3 赋给 double 变量 c，变成 3.0。要得到 3.333...，需要写成 (double)a/b 或 a/3.0。


题目 3：关于 if-else 语句，以下说法正确的是？

A. if-else 语句中，if 和 else 两个分支可能同时执行
B. else 后面必须跟着一个条件表达式
C. if-else 是一个二选一的结构，if 和 else 两个分支必走其一
D. if-else 语句不能嵌套使用

正确答案：C

解析：
if-else 是最简单的分支语句，条件成立走 if 分支，不成立走 else 分支，必走其一且只走其一。else 后面不能跟条件，if-else 可以嵌套。


题目 4：以下哪个条件表达式能正确判断"x 是偶数"？

A. if (x = 0)
B. if (x / 2 == 0)
C. if (x % 2 == 0)
D. if (x % 2 = 0)

正确答案：C

解析：
偶数的特征是除以 2 余数为 0，用 % 求余。A 是赋值不是判断；B 不准确（只有 x=0 或 1 时才成立）；D 混淆了 == 和 =。


题目 5：if (x > 0); 这一行代码中，"if (x > 0);"是语法错误吗？

A. 是语法错误，编译器会报错
B. 不是语法错误，但逻辑上是错误的
C. 这是正确的写法
D. if 不能单独和分号一起用

正确答案：B

解析：
if (x > 0); 在语法上完全合法——它是一个 if 后面跟了一个空语句。编译器不会报错。但这几乎肯定是编程者不小心多写的分号，逻辑上是一个 bug。


题目 6：以下关于运算精度规则的说法，正确的是？

A. 表达式中有 int 时，结果一定是 int
B. char 类型不能参与算术运算
C. 表达式中有 double 时，整个表达式按 double 计算
D. float 和 double 一起运算时，结果按 float 计算

正确答案：C

解析：
有 double 就按 double 算（向上转换规则）。A 错：有 double 时结果就不是 int。B 错：char 可以参与运算且自动转 int。D 错：float 和 double 一起时，float 提升为 double。
```

## 10. 拖拽匹配素材

```text
1. 运算精度规则 —— 表达式中有 double 按 double 算，有 float 按 float 算，全 int 按 int 算
2. 向上转换 —— 低精度类型在混合运算中自动提升为高精度类型
3. 逻辑非 ! —— 反转布尔值，!true 变 false，!false 变 true
4. if 语句 —— 条件判断语句：条件为 true 执行代码块，为 false 跳过
5. if-else 语句 —— 二选一分支语句：条件为 true 走 if 分支，为 false 走 else 分支
6. 分支语句 —— 提供两条（或多条）不同的执行路径，根据条件选择其中一条
7. == 比较运算符 —— 判断两个值是否相等，结果为 boolean 类型
8. int / int —— 整数除法，结果仍为整数，小数部分直接截断
9. int / double —— 混合除法，int 自动转为 double，结果保留小数
10. 代码规范 —— System（S大写）、main（全小写）、println（全小写），严格区分大小写
```

## 11. 流程动画素材

```text
if-else 分支语句的执行流程：

1. 程序从上到下执行，遇到 if 关键字。
2. 计算 if 后面括号中的条件表达式，得到 true 或 false。
3. 如果结果是 true → 进入 if 后面的花括号 { }，执行 if 分支中的所有代码。
4. if 分支执行完毕后 → 跳过 else 分支，直接执行 if-else 结构后面的代码。
5. 如果结果是 false → 跳过 if 分支，进入 else 后面的花括号 { }，执行 else 分支中的所有代码。
6. else 分支执行完毕后 → 继续执行 if-else 结构后面的代码。
7. 无论如何，if 和 else 两个分支只有一个会被执行，然后程序继续向下运行。
```

```text
(2) 8.0 / 5
(3) 8 % 5
(4) 'C' + 1
(5) (double)(8 / 5)

public class LeapYear {
```

```text
1. 本节课的核心是掌握"运算精度规则"和"if 分支语句"两个部分——前者决定了混合运算的结果类型，后者让程序能根据条件做出不同反应。
2. 学习流程控制时要特别注意：if 括号里必须是 boolean 类型、== 是比较而 = 是赋值、if 后面不要多写分号。这几个看似简单的细节是初学者出错最多的地方。
3. 通过本节课，应该能够正确使用 if-else 处理二选一的判断场景，并理解不同精度类型在混合运算中的自动提升行为。
4. if-else 是后续学习循环（for、while）和多路分支（switch）的基础。只有掌握了 if-else，后续的复合控制结构才能顺利理解。
5. 课后建议多练习：(1) 用 if-else 写几个判断程序（奇偶数、正负数、闰年）；(2) 验证运算精度规则——故意写几个混合类型的表达式并输出结果；(3) 用 if-else 做一个简单的"根据分数输出等级"的程序；(4) 预习实验课的 5 个题目（水仙花数、最大公约数等）。
```
