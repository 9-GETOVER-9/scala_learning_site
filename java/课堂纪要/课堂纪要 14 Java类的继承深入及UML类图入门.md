Java类的继承及相关课程安排

【智能总结】

本次课堂讲解Java类的继承知识，包括概念、语法等，还介绍了后续课程安排和UML类图。

\- \*\*继承的概念\*\*：子类由父类派生而来，继承父类非私有属性和方法，且子类只能有一个父类，否则会产生二义性。

\- \*\*继承的语法结构\*\*：使用关键字“extend”定义子类，如“class 子类名 extend 父类名” 。

\- \*\*继承关系的结构\*\*：Java类按继承关系形成树形结构，类似倒置二叉树，每个子节点有且仅有一个父类。

\- \*\*同一包下子类与父类的关系\*\*：在同一包中，子类继承父类非private成员变量和方法，可通过访问器访问父类私有属性。

\- \*\*访问权限\*\*：子类继承父类方法时，成员访问权限保持不变，如private、public、protected和default的可见性不同。

\- \*\*子类对象的创建\*\*：子类构造方法总会先调用父类的某个构造方法。

\- \*\*后续课程安排\*\*：

\- \*\*第九周周一\*\*：线上课，需带作业本，要写作业并进行小测验。

\- \*\*第十周\*\*：讲完最后一章，做最后一个实验，带笔记本，检查笔记送成绩，完成第二次实验报告。

\- \*\*第十一周周一上午\*\*：复习。

\- \*\*第十二周\*\*：周一或周四考试，具体与学习委员商量 。

\- \*\*UML类图\*\*：是面向对象设计的方法，用于需求分析和项目设计，建议自学类图相关内容。
