知识点 1：数组的维度概念（一维、二维、三维）

这个知识点是什么？

数组的维度（dimension）描述了一个数组有几个"方向"。维度越多，数据结构越复杂：

  • 一维数组：只有一个方向，就像一条线（射线），通过一个下标就能访问元素。
  • 二维数组：有两个方向（X 轴和 Y 轴），就像一张表格/矩阵，有行和列，需要两个下标来定位。
  • 三维数组：有三个方向，就像一个立体空间（立方体），需要三个下标来定位。
为什么要学它？

不同维度对应不同的数据组织方式。成绩单是二维的（学生 × 科目），RGB 图像是三维的（宽 × 高 × 颜色通道）。理解维度概念是掌握复杂数据结构的基础。

基本语法
// 一维数组
int[] arr1 = new int[5];           // 像一个有 5 个格子的横排

// 二维数组
int[][] arr2 = new int[3][4];      // 3 行 × 4 列，像一个表格

// 三维数组
int[][][] arr3 = new int[2][3][4]; // 2 层 × 3 行 × 4 列
课堂中的理解方式

老师用几何概念来类比：

  • 一维："一个方向，就像射线。只有一个下标——boy0、boy1、boy2、boy3。"
  • 二维："不仅有 X 轴，还有 Y 轴。有两个值了——前面这个叫行，后面这个叫列。A1 就是第一行第一列。"
  • 三维："再有一个就是三维了。"（课堂上老师用立体空间的概念简单带过）。

老师特别强调：一维数组用一个下标访问（如 arr[0]），二维数组用两个下标（如 arr[行][列]），前面的下标是行，后面的是列。

初学者容易犯的错误
1. 二维数组的行和列搞反——arr[行][列]，不是 arr[列][行]。
2. 访问二维数组时只写了一个下标——二维数组必须用两个下标。
3. 分不清什么场景该用几维数组——能用一个维度解决的不要用两个维度。
4. 下标从 0 开始但习惯从 1 开始数——长度为 4 的数组，下标范围是 0~3。
正确示例
public class ArrayDimensionDemo {
    public static void main(String[] args) {
        // 一维数组：就像一个列表
        String[] boys = {"boy0", "boy1", "boy2", "boy3"};
        System.out.println("第1个：" + boys[0]);  // boy0
        System.out.println("第3个：" + boys[2]);  // boy2

        // 二维数组：就像一张表格（3行 × 3列）
        int[][] matrix = {
            {1, 2, 3},    // 第 1 行
            {4, 5, 6},    // 第 2 行
            {7, 8, 9}     // 第 3 行
        };
        System.out.println("第1行第1列：" + matrix[0][0]);  // 1
        System.out.println("第2行第3列：" + matrix[1][2]);  // 6
    }
}
示例解释
1. boys 是一维数组，只需要一个下标就能定位到元素。
2. matrix 是二维数组，需要两个下标——第一个定位行，第二个定位列。
3. matrix[1][2] 表示第 2 行（下标 1）第 3 列（下标 2）。
4. 二维数组可以理解为"数组的数组"——每一行本身是一个一维数组。

知识点 2：数组的 length 属性

这个知识点是什么？

length 是数组的一个属性（不是方法），表示数组中元素的个数。对于一维数组，length 就是元素的总数；对于二维数组，数组名.length 表示行数，数组名[行下标].length 表示该行的列数。

为什么要学它？

在遍历数组时，必须知道数组有多长才能避免下标越界。length 属性是访问数组时最常用的工具之一。

基本语法
数组名.length           // 一维：元素总个数；二维：行数
数组名[行下标].length   // 二维：该行的列数
课堂中的理解方式

老师用一个关键例子帮助理解：

  • "A1.length 是不是 3？A2.length 是不是也是 3？A3.length 是不是也是 3？"
  • 为什么是 3？因为每一行有 3 个元素（3 列）。
  • A1 是第一行，A1.length 得到的是这一行的列数。
  • 所以二维数组中：数组名.length = 行数，数组名[i].length = 第 i 行的列数。
初学者容易犯的错误
1. 把 length 当成方法用，写了括号：arr.length() —— 正确是 arr.length（没有括号）。
2. 在二维数组中，把 arr.length 理解成总元素个数——实际上 arr.length 是行数。
3. 访问不存在的行的 length：如果只有 3 行，访问 arr[3].length 会下标越界。
正确示例
public class LengthDemo {
    public static void main(String[] args) {
        int[] arr1 = {10, 20, 30, 40, 50};
        System.out.println("一维数组长度：" + arr1.length);   // 5

        int[][] arr2 = {
            {1, 2, 3},      // 第 1 行，3 列
            {4, 5, 6, 7},   // 第 2 行，4 列
            {8, 9}           // 第 3 行，2 列
        };
        System.out.println("二维数组行数：" + arr2.length);       // 3
        System.out.println("第1行列数：" + arr2[0].length);      // 3
        System.out.println("第2行列数：" + arr2[1].length);      // 4
        System.out.println("第3行列数：" + arr2[2].length);      // 2
    }
}
示例解释
1. arr1.length 返回 5——这个一维数组有 5 个元素。
2. arr2.length 返回 3——这个二维数组有 3 行。
3. arr2[0].length 返回 3——第 1 行有 3 列。
4. 注意：Java 的二维数组每行的列数可以不同（不规则数组），arr2[1].length = 4，arr2[2].length = 2。

知识点 3：数组的初始化方式

这个知识点是什么？

数组在声明后，需要初始化才能使用。初始化有两种方式：

  • 方式一（先声明，再赋值）：用 new 分配空间，系统自动给每个元素赋默认值，然后再逐个赋值。
  • 方式二（声明时直接初始化）：用花括号 {} 把初始值一次性列出，简洁方便。

此外，创建数组后系统会给出默认值：int 默认 0，double 默认 0.0，boolean 默认 false，引用类型默认 null。

为什么要学它？

初始化数组是最基本的操作。两种方式各有适用场景——方式一适合"先占位，后填值"，方式二适合"一开始就知道所有值"。

基本语法
// 方式一：先 new，再逐个赋值
类型[] 数组名 = new 类型[长度];
数组名[0] = 值1;
数组名[1] = 值2;
...

// 方式二：声明时花括号初始化
类型[] 数组名 = {值1, 值2, 值3, ...};
课堂中的理解方式

老师特别强调了方式二中方括号里不能写数字："float B[]，方括号里面没有数值。你不能写个 4——你怎么知道后面是第几个呀？有很多同学犯错误，搁这直接就给我写个数，那就不对了。"

换句话说，如果写成 float B[4] = {1, 2, 3, 4}，方括号里的 4 是多余的。应该写成 float[] B = {1, 2, 3, 4}，让 Java 自动根据花括号里的数量推断长度。

老师还指出："按照这个顺序，一个一个地写到我们的 B 的数组里边。第一个就是 B[0]，第二个是 B[1]……"

默认值：老师说明"创建数组后，系统会给数组当中的每一个元素默认一个值。整型就是 0，float 就是 0.0。"

初学者容易犯的错误
1. 在花括号初始化的方括号中写数字：int[5] arr = {1,2,3,4,5};（错误——方括号里不能写数字）。
2. 忘记 new 关键字：int[] arr = int[5];（错误——应该用 new int[5]）。
3. 数组创建后没有赋值就直接使用——值为默认值（0），可能不是想要的。
4. 花括号初始化时多写了分号：int[] arr = {1,2,3,};（最后一个逗号在某些 Java 版本中允许，但不规范）。
正确示例
public class ArrayInitDemo {
    public static void main(String[] args) {
        // 方式一：先 new 再逐个赋值
        int[] scores = new int[5];   // 创建长度为 5 的数组，默认值全是 0
        scores[0] = 90;
        scores[1] = 85;
        scores[2] = 78;
        scores[3] = 92;
        scores[4] = 88;

        // 方式二：花括号直接初始化（方括号里不写数字）
        float[] prices = {9.9f, 19.9f, 29.9f, 39.9f};
        // 等价于：
        // float[] prices = new float[4];
        // prices[0] = 9.9f; prices[1] = 19.9f; ...

        // 验证默认值
        int[] empty = new int[3];
        System.out.println("默认值：" + empty[0]);  // 0

        // 输出结果
        System.out.println("scores[0] = " + scores[0]);   // 90
        System.out.println("prices[2] = " + prices[2]);   // 29.9
    }
}
示例解释
1. scores 使用方式一：先 new int[5] 分配 5 个位置（默认全是 0），再逐个赋值为具体分数。
2. prices 使用方式二：花括号 {} 一次性初始化，方括号里不需要写长度。
3. empty 数组创建后没有赋值，输出 0——这是 int 类型的默认值。
4. 方式一适合"后面才知道具体值"的场景；方式二适合"一开始就知道所有值"的场景。

知识点 4：数组是引用类型——"一处变，处处变"

这个知识点是什么？

Java 中，数组属于引用类型（Reference Type）。数组变量存储的不是数据本身，而是数据在内存中的地址。当两个数组变量指向同一块地址时，通过任意一个变量修改数据，另一个变量看到的也会跟着改变。这就是——一处变，处处变。

为什么要学它？

不理解引用类型，就会出现"明明只改了 A，为什么 B 也跟着变了"的困惑。引用类型是理解 Java 内存模型、方法参数传递的关键。

基本语法
int[] a = {1, 2, 3};
int[] b = a;          // b 指向 a 的同一块地址（不是复制）
b[0] = 100;           // 修改 b 的元素
// 此时 a[0] 也是 100！因为 a 和 b 指向同一地址
课堂中的理解方式

老师用了两个非常精彩的比喻：

比喻一：班长 = 高梦涵

  • "高梦涵"是她的全名，"班长"是她的身份/引用。两个称呼指向的是同一个人。
  • 当座位上的人从高梦涵换成了杜涵蕊，"一班班长"这个称呼指向的人也跟着变了。
  • "引用型变量与原变量指向同一块地址。当地址里的数值变了，那这两个值都变了。"

比喻二：小名（昵称）

  • 一个人有大名（张三）和小名（小小），两个名字指的都是同一个人。
  • "你一个另一个称呼，都是一个人。"
  • "引用，换句话说，就是他的小名。"

核心结论（老师让学生记住的口诀）："一处变，处处变。"

初学者容易犯的错误
1. 以为 int[] b = a; 是复制了一份新数组——实际上是让 b 指向和 a 同样的地址。
2. 修改 b 的元素后，惊讶地发现 a 的元素也变了。
3. 把普通变量的赋值逻辑（a=1; b=a; b=2; 此时 a 还是 1）套用到数组上。
4. 想要复制数组却用 = 赋值——应该用循环逐个复制，或用 clone() / System.arraycopy()。
正确示例
public class ReferenceDemo {
    public static void main(String[] args) {
        // ====== 对比 1：普通变量（值传递） ======
        int x = 10;
        int y = x;        // y 复制了 x 的值（10），但它们是独立的
        y = 20;           // 修改 y
        System.out.println("x = " + x);  // x 还是 10（不受影响）
        System.out.println("y = " + y);  // y 是 20

        // ====== 对比 2：数组（引用传递） ======
        int[] arrA = {1, 2, 3};
        int[] arrB = arrA;      // arrB 指向 arrA 的同一块地址！
        arrB[0] = 100;          // 通过 arrB 修改

        System.out.println("arrA[0] = " + arrA[0]);  // 100！（也跟着变了）
        System.out.println("arrB[0] = " + arrB[0]);  // 100

        // ====== 正确复制数组的方法 ======
        int[] source = {10, 20, 30};
        int[] copy = new int[source.length];
        for (int i = 0; i < source.length; i++) {
            copy[i] = source[i];   // 逐个复制元素
        }
        copy[0] = 999;  // 修改 copy 不影响 source
        System.out.println("source[0] = " + source[0]);  // 10（没变）
        System.out.println("copy[0] = " + copy[0]);      // 999
    }
}
示例解释
1. int y = x; 是值传递——y 获得了 x 的值（10），之后 x 和 y 各自独立。修改 y 不影响 x。
2. int[] arrB = arrA; 是引用传递——arrB 获得了 arrA 存储的地址，两个变量指向同一块内存。
3. arrB[0] = 100; 修改了该地址的第一个元素，arrA[0] 自然也变了（因为它们看的是同一块地方）。
4. 要真正复制一个数组，需要创建一个新数组，然后用循环逐个复制元素。

知识点 5：普通变量 vs 引用型变量

这个知识点是什么？

Java 中的变量分为两类：

  • 普通变量（基本类型）：直接存储数据值本身（如 int、double、boolean）。赋值时复制的是值，两个变量互不影响。
  • 引用型变量（引用类型）：存储的是数据在内存中的地址（如数组、String、自定义类的对象）。赋值时复制的是地址，两个变量指向同一个数据。
为什么要学它？

这是 Java 中最容易被忽视但影响深远的概念。不理解这个区别，后面学方法参数传递（值传递 vs 引用传递）、对象比较（== vs equals）都会经常出错。

基本语法
// 普通变量：值传递
int a = 1;
int b = a;   // b 复制 a 的值
a = 2;       // a 变了，b 不变

// 引用变量：地址传递
int[] arr1 = {1, 2};
int[] arr2 = arr1;   // arr2 复制 arr1 的地址
arr1[0] = 100;       // arr1 和 arr2 都看到变化
课堂中的理解方式

老师用一个清晰的对比来解释：

  • 普通变量："A 变 B 不变，你愿意怎么变就怎么变，跟我没有关系。A 怎么变 B 也不管，B 怎么变 A 也不管。这是两个不同的东西。"
  • 引用变量："B 变 A 就变，A 变 B 就变。这两个东西都指向同一块地址，就是一个东西。"

老师还打了一个数学的比方："就像 e 的 a 次方和 e 的 b 次方一样，在某一时刻它俩都等于 1。过一段时间 a 变成 2 了，但 b 还是一。这就是两个不同的变量。"

初学者容易犯的错误
1. 把数组赋值当成"复制内容"——实际上复制的是地址。
2. 在写方法时，把数组作为参数传入，在方法内部修改数组，结果方法外面的数组也变了。
3. 对 String 类型的引用行为产生误解——String 虽然是引用类型，但它是不可变的（后面会学）。
正确示例
public class ValueVsReference {
    public static void main(String[] args) {
        // ====== 普通变量（值传递）======
        int num1 = 10;
        int num2 = num1;      // 复制值
        num1 = 999;
        System.out.println("普通变量：");
        System.out.println("num1 = " + num1);  // 999（num1 自己变了）
        System.out.println("num2 = " + num2);  // 10（num2 不受影响）

        // ====== 引用变量（地址传递）======
        int[] arr1 = {10, 20};
        int[] arr2 = arr1;    // 复制地址（不是复制内容）
        arr1[0] = 999;
        System.out.println("\n引用变量：");
        System.out.println("arr1[0] = " + arr1[0]);  // 999
        System.out.println("arr2[0] = " + arr2[0]);  // 999（也跟着变了！）
    }
}
示例解释
1. num1 和 num2 各存各的值。num1 改成 999 完全不关 num2 的事。
2. arr1 和 arr2 存的是同一个地址。arr1[0] 改成 999，arr2 看到的也是改过的数据——因为它们"看的是同一块地方"。
3. 这就像一个班级的花名册有两份复印件——在一份上改了一个名字，另一份不会变（值传递）；但如果两个都是指向同一本花名册的"快捷方式"，那么修改花名册本身，两个快捷方式看到的内容都变了（引用传递）。

知识点 6：数组默认值与逐个赋值

这个知识点是什么？

当用 new 类型[长度] 创建数组后，在没有手动赋值之前，Java 会自动给每个元素一个默认值：

数组类型 默认值
int[] 0
double[] 0.0
float[] 0.0f
boolean[] false
char[] ''（空字符）
引用类型[]（如 String[]） null
为什么要学它？

如果不了解默认值机制，就可能误以为未赋值的数组元素是"空的"或"随机的"，导致逻辑 bug。

基本语法
int[] arr = new int[5];     // 5 个元素默认全是 0
arr[0] = 10;                // 只给第 1 个赋值
// arr[1] ~ arr[4] 仍然是 0
课堂中的理解方式

老师用一句话总结："创建数组后，系统会给数组当中的每一个元素默认一个值。整型就是 0。"

然后说明了赋值的两种途径：

  1. 逐个赋值："要 1 给他，2 给他，他他他一个一个给。"
  2. 初始化列表："按照这个顺序，一个一个地写到数组里边。第一个就是 B[0]，第二个 B[1]……"
初学者容易犯的错误
1. 以为 new int[5] 后元素的值是"空"或"未定义"——实际全是 0。
2. 以为 new String[5] 后元素是空字符串 ""——实际是 null（null 和 "" 完全不同）。
3. 期望默认值是随机数——Java 会明确赋默认值，不会有"垃圾数据"。
正确示例
public class DefaultValueDemo {
    public static void main(String[] args) {
        int[] intArr = new int[3];
        double[] doubleArr = new double[3];
        boolean[] boolArr = new boolean[3];
        String[] strArr = new String[3];

        System.out.println("int 默认值：" + intArr[0]);       // 0
        System.out.println("double 默认值：" + doubleArr[0]); // 0.0
        System.out.println("boolean 默认值：" + boolArr[0]);  // false
        System.out.println("String 默认值：" + strArr[0]);    // null

        // 注意区分 null 和 ""
        // strArr[0].length();  ← 这会报错（NullPointerException），因为 strArr[0] 是 null
        // "".length();         ← 这没问题，返回 0（空字符串不是 null）
    }
}
示例解释
1. 每种类型的数组创建后都有默认值，不是"随机"的。
2. int 默认 0，double 默认 0.0，boolean 默认 false。
3. String 是引用类型，默认值是 null（啥也没有），不是空字符串 ""。
4. null 表示"没有指向任何对象"，对 null 调用方法会抛出 NullPointerException。

概念 A 概念 B 区别 示例

6. 代码示例整理

示例 1：一维数组与二维数组的 length 属性

适用知识点：

length 属性、一维数组、二维数组、行数列数

代码：

public class LengthDemo {
    public static void main(String[] args) {
        // 一维数组
        double[] grades = {85.5, 92.0, 78.5, 88.0};
        System.out.println("成绩数组长度：" + grades.length);       // 4

        // 二维数组（不规则：每行列数可以不同）
        int[][] table = {
            {1, 2, 3},          // 第 1 行：3 列
            {4, 5},             // 第 2 行：2 列
            {6, 7, 8, 9}        // 第 3 行：4 列
        };

        System.out.println("table 行数：" + table.length);          // 3
        for (int i = 0; i < table.length; i++) {
            System.out.println("第" + (i+1) + "行列数：" + table[i].length);
        }
        // 输出：
        // 第1行列数：3
        // 第2行列数：2
        // 第3行列数：4
    }
}

运行结果：

成绩数组长度：4
table 行数：3
第1行列数：3
第2行列数：2
第3行列数：4

代码解释：

1. grades.length 返回 4，因为一维数组有 4 个元素。
2. table.length 返回 3，因为二维数组有 3 行。
3. table[i].length 返回第 i 行的列数——每行的列数可以不同（不规则数组）。
4. 第 1 行有 {1,2,3} 3 个元素，第 2 行有 {4,5} 2 个元素。

初学者提示：

- length 是属性，没有括号——写 arr.length() 会报错！
- 二维数组每一行的列数不一定要相同，这叫"不规则数组"（Jagged Array）。

示例 2：普通变量 vs 引用型变量的区别

适用知识点：

值传递、引用传递、数组引用

代码：

public class ValueVsReference {
    public static void main(String[] args) {
        // ====== 场景 1：普通变量 ======
        System.out.println("===== 普通变量 =====");
        int a = 10;
        int b = a;          // b 复制了 a 的值
        System.out.println("赋值后：a=" + a + ", b=" + b);  // a=10, b=10

        a = 999;            // 修改 a
        System.out.println("修改 a 后：a=" + a + ", b=" + b);  // a=999, b=10（b 不变！）

        // ====== 场景 2：数组（引用类型） ======
        System.out.println("\n===== 数组（引用类型） =====");
        int[] arr1 = {10, 20, 30};
        int[] arr2 = arr1;  // arr2 指向 arr1 的同一块地址
        System.out.println("赋值后：arr1[0]=" + arr1[0] + ", arr2[0]=" + arr2[0]);

        arr1[0] = 999;      // 通过 arr1 修改
        System.out.println("修改 arr1[0] 后：arr1[0]=" + arr1[0] + ", arr2[0]=" + arr2[0]);
        // arr1[0]=999, arr2[0]=999（arr2 也跟着变了！）
    }
}

运行结果：

===== 普通变量 =====
赋值后：a=10, b=10
修改 a 后：a=999, b=10

===== 数组（引用类型） =====
赋值后：arr1[0]=10, arr2[0]=10
修改 arr1[0] 后：arr1[0]=999, arr2[0]=999

代码解释：

1. 普通变量 a=10, b=a 后：a 和 b 各存各的。改 a 不影响 b。
2. 数组 arr2 = arr1 后：arr2 拿到的是 arr1 的地址，它们指着同一块内存。
3. 所以改 arr1[0] = 999 后，arr2[0] 也变成了 999——"一处变，处处变"。

初学者提示：

- 口诀：基本类型赋值是"复印"（各拿一份）；引用类型赋值是"发定位"（指向同一个地方）。
- 想要真的复制数组内容，必须 new 一个新数组，然后循环逐个复制。

示例 3：数组初始化的正确与错误写法

适用知识点：

数组初始化、默认值、花括号语法

代码：

public class ArrayInitDemo {
    public static void main(String[] args) {
        // ✅ 正确写法 1：先 new，再逐个赋值
        int[] scores = new int[3];
        scores[0] = 90;
        scores[1] = 85;
        scores[2] = 78;

        // ✅ 正确写法 2：花括号直接初始化（方括号里不写数字）
        float[] prices = {9.9f, 19.9f, 29.9f};

        // ✅ 正确写法 3：声明和初始化分开
        String[] names;
        names = new String[]{"张三", "李四", "王五"};  // 分开写时前面要加 new String[]

        // ❌ 错误写法 1：花括号初始化时方括号里写了数字
        // int[3] arr = {1, 2, 3};   // 编译错误！

        // ❌ 错误写法 2：分开声明时忘记 new String[]
        // String[] names;
        // names = {"张三", "李四", "王五"};  // 编译错误！

        // 验证结果
        System.out.println("scores[0] = " + scores[0]);    // 90
        System.out.println("prices[1] = " + prices[1]);    // 19.9
        System.out.println("names[2] = " + names[2]);      // 王五
    }
}

运行结果：

scores[0] = 90
prices[1] = 19.9
names[2] = 王五

代码解释：

1. 方式一（new + 逐个赋值）：适合先占位后填值的场景。
2. 方式二（花括号）：适合一开始就知道所有值的场景，方括号里不能写数字。
3. 方式三（声明与初始化分开）：分开时，赋值的右边必须写 new String[]{...}，不能只写 {...}。
4. 错误写法 1：int[3] arr = {1,2,3}; 方括号里写了 3——Java 会根据花括号里的数量自动推断。
5. 错误写法 2：分开声明时直接写 {...}——Java 不允许，必须加 new String[]。

初学者提示：

- 记住口诀：花括号初始化，方括号里不写数。
- 声明和初始化写在同一行 → 可以用 {...}。
- 声明和初始化分开写 → 必须用 new 类型[]{...}。

案例 1：班长与高梦涵——理解引用类型

  • 案例名称：为什么"一班班长"换了？
  • 对应知识点：引用类型、地址传递
  • 案例背景：老师为了解释引用类型的概念，让班长高梦涵站起来，用她的真实姓名和"班长"这个职务称呼做对比。
  • 解决思路：
    1. "高梦涵"是她的全名——相当于数据的实际内容。
    2. "一班班长"是一个引用——指向坐在那个座位上的人。
    3. 当座位上的人从高梦涵换成了杜涵蕊，"班长"这个引用指向的人也自动变了。
    4. 两个称呼（高梦涵和一班班长）指向的是同一块地址（同一个座位）。
  • Java 代码或伪代码：见第 6 节示例 2。
  • 这个案例帮助理解什么：直观地说明了"引用不是数据本身，而是指向数据的一个地址。当数据变了，通过任何一个引用看到的结果都会变。"

案例 2：小名（昵称）——另一个引用型变量的类比

  • 案例名称：张三和小小都是同一个人
  • 对应知识点：引用型变量、一处变处处变
  • 案例背景：老师用"小名"来做第二个比喻，帮助学生进一步理解引用。
  • 解决思路：一个人有大名（张三）和小名（小小），但指的都是同一个人。数组的两个引用变量（arr1 和 arr2）就像大名和小名——通过哪个名字来改，改的都是同一个人。
  • 这个案例帮助理解什么：强化"引用型变量 = 指向同一块数据的别名"这个概念。

8. 易错点总结

易错点 1：把 length 当方法，写成了 arr.length()
错误示例：
int len = arr.length();    // 编译错误！

正确写法：
int len = arr.length;      // length 是属性，没有括号

解释：
数组的 length 是一个属性（property），不是方法（method）。只有 String 的 length 才是方法（str.length()），不要混淆。


易错点 2：以为 int[] b = a; 是复制了新数组
错误示例：
int[] a = {1, 2, 3};
int[] b = a;
b[0] = 100;     // 以为只改了 b，实际上 a[0] 也变成了 100

正确做法（要真正复制）：
int[] b = new int[a.length];
for (int i = 0; i < a.length; i++) {
    b[i] = a[i];
}

解释：
int[] b = a; 只是让 b 也指向 a 的那块内存，两个变量共享同一份数据。要复制内容必须 new 新数组 + 循环复制。


易错点 3：花括号初始化时在方括号里写数字
错误示例：
int[5] arr = {1, 2, 3, 4, 5};   // 编译错误

正确写法：
int[] arr = {1, 2, 3, 4, 5};     // 方括号里不写数字

解释：
花括号里的值已经说明了数组的元素个数，Java 会自动推断长度。在方括号里再写数字是多余的，语法不允许。


易错点 4：二维数组行列下标搞反
错误示例：
int[][] arr = new int[3][4];
arr[0][2]   // 以为这是第 2 行第 0 列

正确理解：
arr[0][2]   // 第 1 行（下标 0）第 3 列（下标 2）
// 格式固定为：arr[行][列]，前面的永远是行，后面的永远是列

解释：
二维数组的下标格式是 [行][列]，不能反过来写。行在前，列在后。


易错点 5：未赋值的数组元素以为是"空"，实际是默认值
错误示例：
int[] arr = new int[5];
if (arr[0] == null) { ... }    // null 不能和 int 比较，编译错误

正确理解：
int[] arr = new int[5];
// arr[0] 的值是 0（不是 null，不是未定义）

解释：
Java 中数组创建后，每个元素都有默认值。int 是 0，String[] 才是 null。

10. 拖拽匹配素材

1. 一维数组 —— 只有一个方向，通过一个下标访问元素，就像一条射线
2. 二维数组 —— 有两个方向（行和列），通过 arr[行][列] 访问，就像一张表格
3. length 属性 —— 获取数组元素个数，一维是总个数，二维是行数
4. 数组默认值 —— 创建数组后系统自动赋予的初始值，int 为 0，double 为 0.0
5. 引用类型 —— 变量存储的是内存地址而非数据本身，数组属于引用类型
6. 值传递 —— 普通变量赋值时复制数值，两个变量互不影响
7. 地址传递 —— 引用变量赋值时复制地址，指向同一块数据的变量相互影响
8. 花括号初始化 —— 声明数组时直接用 {} 给出所有值，方括号里不写数字
9. null —— 引用类型的默认值，表示"没有指向任何对象"
10. 一处变处处变 —— 引用型变量的核心特征：通过任一引用修改数据，所有引用都能看到变化

11. 流程动画素材

引用类型变量的工作原理：

1. 程序执行 int[] a = {1, 2, 3}; —— JVM 在内存中分配一块空间，存入 {1, 2, 3}，变量 a 中存储的是这块空间的地址。
2. 程序执行 int[] b = a; —— 变量 b 获得了 a 中存储的地址，现在 a 和 b 指向同一块内存空间。
3. 程序执行 b[0] = 100; —— JVM 根据 b 中的地址找到内存空间，将第一个元素改为 100。
4. 此时通过 a 查看 —— a 也指向同一块空间，所以 a[0] 也是 100。
5. 这就是"一处变，处处变"——改的是内存中的数据本身，所有指向这块内存的引用都能看到变化。
6. 如果这是普通变量 int x = 10; int y = x; y = 20; —— y 变 20 不影响 x，因为它们是各自独立的内存空间。

练习 3：阅读以下代码，说明为什么 arr1[0] 的值变了。

int[] arr2 = arr1;

练习 4：写一个程序，先创建一个包含 {10, 20, 30, 40, 50} 的数组 a，然后创建一个新数组 b，将 a 的内容真正复制到 b 中（不是引用复制），然后修改 b[0] = 999，验证 a[0] 是否还是 10。

考查知识点：
数组复制（非引用）、值传递概念

参考答案：

public class ArrayCopy {
    public static void main(String[] args) {
        int[] a = {10, 20, 30, 40, 50};

        // 真正复制：new 新数组 + 逐个元素复制
        int[] b = new int[a.length];
        for (int i = 0; i < a.length; i++) {
            b[i] = a[i];
        }

        b[0] = 999;    // 修改 b

        System.out.println("a[0] = " + a[0]);  // 10（没变！）
        System.out.println("b[0] = " + b[0]);  // 999
    }
}

解题思路：
想要真正的复制（两个独立的数组），必须 new 一个新数组，然后循环复制每个元素。直接用 = 赋值只是引用传递，不是复制。


练习 5：找出以下代码中的 4 处错误。

public class Test {
    public static void main(String[] args) {
        int[3] arr = {1, 2, 3};            // 错误1
        int len = arr.length();             // 错误2
        int[][] matrix = new int[2][2];
        matrix[1][1] = 10;
        System.out.println(matrix[2][0]);   // 错误3
    }
}

考查知识点：
数组初始化语法、length 属性、下标边界

参考答案：
错误1：int[3] arr → int[] arr（花括号初始化时方括号不能写数字）
错误2：arr.length() → arr.length（length 是属性，没有括号）
错误3：matrix[2][0] → 行下标越界，matrix 只有 2 行（下标 0 和 1），[2] 访问的是第 3 行

解题思路：
逐行检查语法：花括号初始化规则 → length 属性无括号 → 下标不能 ≥ 数组长度。

15. 课后总结

1. 本节课的核心是掌握数组的维度概念（一维/二维/三维）、length 属性的用法，以及区分数组初始化的两种方式（new + 逐个赋值 vs 花括号一次性初始化）。
2. 学习数组时要特别注意两个容易出错的点：花括号初始化时方括号里不能写数字，length 是属性不是方法（没有括号）。
3. 本节课最重要的概念是引用类型——数组是引用类型变量，int[] b = a; 复制的是地址而不是内容。记住口诀："一处变，处处变。"
4. 引用类型的概念在后续的方法参数传递、对象赋值、集合操作中会反复出现，是 Java 内存模型的核心，一定要理解透彻。
5. 课后建议多练习：(1) 用两种方式初始化数组并输出；(2) 创建一个二维数组并用双重循环输出表格；(3) 验证引用类型的行为——写代码对比普通变量赋值和数组赋值的结果差异；(4) 实现数组的"真正复制"。
