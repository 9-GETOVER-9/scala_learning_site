课程总结

1. 课程基本信息

  • 课程 ID 建议：java-oop-review-access
  • 课程标题：类与对象综合复习——变量、方法与访问权限
  • 所属章节：Java 类与对象
  • 上课日期：2025-10-09
  • 知识点标签：类定义 成员变量 局部变量 构造方法 实例成员 类成员 static 访问权限 public private protected 包
  • 难度等级：巩固-进阶
  • 适合对象：已学完类的全部内容、需要考前综合复习的初学者
  • 本节课一句话概括：本节课是类与对象章节的全面复习课——系统梳理了类的结构、成员变量/局部变量、构造方法重载、实例成员与类成员的区别、参数传值（值传递/址传递）、以及四种访问权限修饰符（public/protected/default/private）。

2. 课堂摘要

1. 本节课首先回顾了类的完整结构——关键字 class、类体（属性 + 方法）、方法的细分（构造方法→访问器→常用方法→特殊方法）。
2. 课堂中重点复习了成员变量与局部变量的区别——成员变量有默认值（系统自动初始化）、局部变量使用前必须初始化，同名时局部变量隐藏全局变量。
3. 老师通过"全班总人数"的类比，帮助学生理解实例成员（属于对象）和类成员（static修饰，属于类，所有对象共享）的区别。
4. 本节课还复习了构造方法重载（参数列表决定构造出什么样的对象）以及对象引用（将一个类的对象作为另一个类的属性——实现任务并发）。
5. 最后通过访问权限修饰符（public/protected/default/private）的讲解，引入了面向对象封装的核心——可见即可访问，通过限制可见性来控制访问权限。

3. 本节课学习目标

1. 能完整说出一个类的结构组成：类声明（class + 类名）→ 属性（成员变量）→ 方法（构造方法→访问器→普通方法）。
2. 能区分成员变量（全局、有默认值）和局部变量（方法内、必须初始化），并说明同名时的隐藏规则。
3. 能区分实例成员（无 static，对象调用）和类成员（有 static，类名调用），并举例说明类变量的共享特性。
4. 能解释构造方法参数列表决定对象特征——不同构造方法创建不同特征的对象（重载）。
5. 能说出值传递（基本类型传副本）和址传递（引用/数组/对象传地址）的区别。
6. 能说出四种访问权限修饰符的名称和基本含义——public（公有）、private（私有）、protected（受保护）、default（默认）。

4. 知识点详细整理

知识点 1：类的完整结构总结

这个知识点是什么？

一个完整的 Java 类包含以下结构层次：

类声明（class + 类名）
└── 类体 { }
    ├── 属性（成员变量）——描述"有什么"
    └── 方法（成员方法）——描述"能做什么"
        ├── 构造方法——创建对象时自动调用，可重载
        ├── 访问器（get/set）——安全地赋值和取值
        ├── 常用方法（common method）——类共有的常规功能
        └── 特殊方法（special method）——类的专项特征
课堂中的理解方式

老师用"衣柜"的类比来解释为什么要按结构组织类——"就像衣柜一样，春夏秋冬的衣服一类一类放。突然下雪了想找厚衣服——你分好类一打开就拿到了；如果乌泱泱一片，找个衣服要翻半天。"

类体的两个构成部分
  • 变量的声明（成员变量）——可以不用赋值语句，系统自动初始化（0/null/false）
  • 方法的定义（成员方法）——成员变量的操作只能在方法体中进行，没有单独的赋值语句

知识点 2：成员变量有默认值，局部变量必须初始化

为什么？

老师用"婴儿出生"的类比：

  • "一个孩子刚出生出来，他就带有自身的属性——皮肤、眼睛、嘴巴、头发。这些是自身带来的，系统会自动给你赋一个初值。"
  • "但后天学习的东西——你说话的能力、学习的能力，系统不管。这是你自己的，必须你自己给出初始值。"
变量类型 默认值 必须初始化
成员变量（全局） 有（0/null/false） 可以不用
局部变量 无 使用前必须初始化

知识点 3：实例成员 vs 类成员（static）

课堂中的类比——全班总人数

老师用"全班总人数"来解释类变量的共享特性：

  • "问 A 同学：你班多少人？54 个。问 B 同学：你班多少人？54 个。不可能 A 同学说 54，B 同学说 53。所以总人数是属于全局的、所有对象共享的。"
  • "突然转来一个同学，变成 55 了——类变量自动更新为 55。A 同学再问也是 55。"
  • "但班长就是一个对象——他能做的一些事情，别人调用不了。那是属于对象的东西。"

核心结论：

  • 有 static → 类成员（类变量 / 类方法）→ 类名直接调用
  • 无 static → 实例成员（实例变量 / 实例方法）→ 对象调用

类方法不能操作实例变量——因为"类方法在对象生成之前就已经存在了，而实例变量在对象 new 出来之后才有。"

知识点 4：构造方法的参数列表决定对象特征

这个知识点是什么？

new 类名(参数) 中括号里的参数必须与某个构造方法的形参列表一一对应。不同参数对应不同的构造方法，构造出来的对象特征也不一样。

课堂中的理解方式

老师用"龙生九子"的比喻：

  • "龙有九子，各有不同。为什么不同？因为参数不一样——有的是跟什么生的，有的是跟什么生的。虽然都是龙的孩子，但特征不一样。"
  • 同样是 people 的对象，new People() 构造的是"没有特征"的普通对象，new People("Tom", 20) 构造的是"有名字、有年龄"的对象。

知识点 5：对象引用——将对象作为另一个类的属性

这个知识点是什么？

一个类可以把另一个类的对象作为自己的属性。这种设计模式可以复用已有功能，实现"任务分发"。

课堂中的理解方式

老师用了一个非常精彩的"外包计算"类比：

  • "我不太会做 1+10 等于几。但我可以让张三做——张三会做 1+10。我把张三作为我的属性，让张三帮我做，把结果告诉我就行了。"
  • "李四会做 10+20，王五会做 20+30。突然接到任务让我算 1+30 的和。我根本不会加法——但我让张三、李四、王五分别算他们会的部分，再把数捏起来告诉我。"
  • "我什么也没干，但我有他们三个作为我的属性——这就叫任务的并发。"

知识点 6：四种访问权限修饰符

基本含义
修饰符 含义 类比
public 公有的——任何地方都能访问 寝室的水壶（谁都能用）
protected 受保护的——同包和子类能访问 兄弟姐妹的财产（有"血缘关系"才能用）
default（不写） 默认的——只有同包能访问 室友的东西（同寝室的可以借）
private 私有的——只有本类能访问 自己的水杯（只有自己能用）
课堂中的理解方式

老师用班级管理做类比：

  • "一班可以调用一些东西，二班可以调用另一些东西，班长可以调用一些，学委可以调用另一些。每个人调用的东西不同。如何分权限？通过修饰符。"

核心原则：可见即可访问——invisible 不可见就不能访问。通过限制可见性来控制访问权限。

5. 重点概念对比

概念 A 概念 B 区别
成员变量（全局） 局部变量 全局有默认值/全程可见；局部无默认值/只在方法内可见
实例方法（无 static） 类方法（有 static） 对象调用 / 类名调用
实例变量 类变量（static） 每个对象各有一份 / 所有对象共享一份
构造方法 普通方法 无返回值类型、名与类同名、创建对象时自动调用
值传递 址传递 基本类型传副本互不影响 / 引用/数组/对象传地址共享修改
public private 任何地方都能访问 / 只有本类能访问

6. 代码示例整理

示例 1：实例成员与类成员的区别

public class Classroom {
    String studentName;            // 实例变量——每个学生有自己的名字
    static int totalCount = 0;    // 类变量——全班共享的总人数

    public Classroom(String name) {
        this.studentName = name;
        totalCount++;             // 每创建一个学生，总人数 +1
    }

    public void showInstance() {
        System.out.println("我叫" + studentName);    // 实例方法
    }

    public static void showTotal() {
        System.out.println("全班共" + totalCount + "人");  // 类方法
    }
}

public class Main {
    public static void main(String[] args) {
        Classroom.showTotal();  // 全班共0人

        Classroom s1 = new Classroom("张三");
        Classroom s2 = new Classroom("李四");
        Classroom s3 = new Classroom("王五");

        Classroom.showTotal();  // 全班共3人——所有学生共享
        s1.showInstance();      // 我叫张三——实例独有的
    }
}

示例 2：对象作为另一个类的属性

// 计算器——会做加法
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    public int addRange(int start, int end) {
        int sum = 0;
        for (int i = start; i <= end; i++) sum += i;
        return sum;
    }
}

// 任务管理器——自己不计算，用计算器来完成
public class TaskManager {
    private Calculator calc = new Calculator();  // 对象作为属性！

    public void doTask() {
        int r1 = calc.add(10, 20);              // 让计算器算 10+20
        int r2 = calc.addRange(1, 30);           // 让计算器算 1~30 的和
        System.out.println("10+20 = " + r1);
        System.out.println("1~30 之和 = " + r2);
    }
}

示例 3：值传递 vs 址传递

public class PassDemo {
    // 值传递——改副本
    public static void changeInt(int n) { n = 999; }

    // 址传递——改原始数据
    public static void changeArray(int[] arr) { arr[0] = 999; }

    public static void main(String[] args) {
        int a = 10;
        changeInt(a);
        System.out.println(a);        // 10（值传递，没变）

        int[] nums = {10, 20};
        changeArray(nums);
        System.out.println(nums[0]);  // 999（址传递，变了！）
    }
}

7. 易错点总结

易错点 1：局部变量未初始化就使用——编译错误。成员变量有默认值，但局部变量没有。
易错点 2：类方法（static）中访问实例变量——编译错误。类方法在对象创建前就存在，无法访问尚未创建的实例变量。
易错点 3：构造方法写了 void——public void 类名() 已经不是构造方法，变成了普通方法。构造方法不能写返回值类型。
易错点 4：包名不对导致找不到类——复制粘贴时忘记改 package 声明。包名和文件夹路径必须一致。
易错点 5：址传递中不小心修改了传入的对象或数组——基本类型安全，但引用类型传递的是地址，方法内的修改会影响外部。

9. 拖拽匹配素材

1. 类（class） —— Java 程序的基本单位，包含属性和方法
2. 成员变量 —— 类体中声明的变量，有默认值，整个类体可见
3. 局部变量 —— 方法体内声明的变量，必须初始化，只在方法内可见
4. 实例方法 —— 无 static 修饰，由对象调用
5. 类方法 —— 有 static 修饰，由类名直接调用
6. 构造方法 —— 与类同名、无返回值、new 时自动调用
7. 重载 —— 同一类中方法名相同、参数列表不同
8. 值传递 —— 基本类型传副本，互不影响
9. 址传递 —— 引用类型传地址，一处变处处变
10. 访问权限 —— 通过修饰符控制谁能访问（public/protected/default/private）
