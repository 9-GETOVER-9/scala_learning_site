2. 课堂摘要

1. 本节课首先明确了方法由方法头（header）和方法体（body）两部分构成，返回值类型由 return 的值的类型决定。
2. 课堂中重点讲解了实例方法与类方法的区别——无 static 的是实例方法（由对象调用），有 static 的是类方法（由类名调用）。
3. 老师通过"同一个模特换不同衣服走 T 台"的生动类比，帮助理解构造方法的重载（Overload）——名称相同，参数列表不同（个数、类型、顺序）。
4. 本节课还讲解了缺省构造方法——即使程序员不写，系统也会自动生成一个无参的构造方法来实例化对象。
5. 最后通过"子晴和高子晴"的类比，深入区分了值传递（传副本，互不影响）和址传递（传地址，一处变处处变）。

3. 本节课学习目标

1. 能说出方法的两大分类——实例方法（无 static，对象调用）和类方法（有 static，类名调用）。
2. 能写出构造方法，并理解构造方法的三个特点：名称与类名相同、无返回值类型、在创建对象时自动调用。
3. 理解缺省构造方法的概念——程序员不写时系统自动提供，不写时也能 new 对象。
4. 能解释方法重载（Overload）的含义：同一类中方法名相同、参数列表不同（个数/类型/顺序——"一同三不同"）。
5. 能区分值传递（基本类型传副本）和址传递（引用类型传地址），并举例说明。

4. 知识点详细整理

知识点 1：实例方法与类方法

这个知识点是什么？

Java 中的方法根据是否有 static 修饰分为两类：

类型 修饰符 由谁调用 特点
实例方法 无 static 对象调用 可以先有类，等对象创建出来再调用
类方法 有 static 类名直接调用 "在对象生成之前就存在了"，类加载时就创建
为什么重要？

理解两者的区别是后续学习 static 关键字、单例模式、工具类设计的基础。用错了调用方式代码会编译报错。

课堂中的理解方式

老师反复强调：

  • "没有 static 的就是实例方法——用实例（对象）来调用。有 static 的就是类方法——用类名来调用。"
  • "类方法是在对象生成之前就存在的。类体结构一写好，类方法就已经创建完了。"
  • 为什么实例变量不能调用类方法？因为"先有类，再 new 出对象。类方法在对象之前就存在了，你总不能让还没出生的对象去做事情吧？"
基本语法
public class MyClass {
    // 实例方法——无 static，由对象调用
    public void instanceMethod() {
        System.out.println("需要对象才能调用");
    }

    // 类方法——有 static，由类名调用
    public static void classMethod() {
        System.out.println("直接由类名调用");
    }
}

// 调用方式
MyClass obj = new MyClass();
obj.instanceMethod();           // 对象调用实例方法
MyClass.classMethod();          // 类名调用类方法

知识点 2：构造方法的特点与缺省构造方法

这个知识点是什么？

构造方法是创建对象时自动调用的特殊方法。它有三个关键特征：

  1. 名称必须与类名完全相同
  2. 没有返回值类型（连 void 也不写）
  3. 在 new 对象时自动调用，用于初始化对象

缺省构造方法：如果程序员没有手动写任何构造方法，Java 会自动提供一个无参数的构造方法（方法体为空），保证 new 类名() 能够执行。

课堂中的理解方式

老师解释缺省构造方法："你不写它，当你创建对象的时候，Java 的 library（类库）会自动创建一个缺省的构造方法。参数列表是空的，body 里面也是空的。然后给对象进行实例化。"

对于有参构造方法，老师强调："参数列表决定了构造出什么样的对象。就像龙有九子，各有不同——为什么不同？因为参数不一样。虽然都是 people 的对象，但对象的特征不一样。"

知识点 3：方法重载（Overload）

这个知识点是什么？

重载（Overload） 是指在同一个类中，允许多个方法拥有相同的名字，但参数列表必须不同。参数列表的不同体现在三个方面——个数、类型、顺序。简称"一同三不同"。

  • 一同：方法名相同
  • 三不同：参数个数不同 / 参数类型不同 / 参数顺序不同
为什么重要？

重载让同一个方法名可以根据不同的参数执行不同的逻辑。比如 System.out.println() 可以接受 int、double、String 等各种类型的参数——这就是重载。

课堂中的理解方式

老师用赵本山小品中的"T 台走秀"作类比：

  • "T 台的人（模特）是一样的，谁在变？衣服在变。人不变，参数变。"
  • person 有三个构造方法——第一个没有参数（穿睡衣），第二个有一个参数（穿内衣/春天），第三个有两个参数（穿大衣/冬天）。
  • "人一直没变，变的是身上的衣服——衣服就是参数。"

老师还特别强调术语——"重载（overload），不是重写（override）。两者都是多态的体现，但重载发生在一个类体内部。"

知识点 4：值传递 vs 址传递

这个知识点是什么？
传递方式 适用类型 特点 类比
值传递 基本数据类型（int、double等） 传递的是值的副本，修改副本不影响原值 把文件复印一份给别人，别人改了复印件不影响原件
址传递 引用类型（数组、对象、String等） 传递的是地址，多个引用指向同一块内存，一处变处处变 把文件的位置（地址）告诉别人，别人直接改原件
课堂中的理解方式

老师用"子晴和高子晴"来解释：

  • "高子晴有一个小名叫子晴。子晴和高子晴都是同一个人。明年子晴长了一岁，高子晴也长了一岁——因为两个名字指向的是同一块地址。这就叫址传递。"
  • "值传递就像两个人交换位置——你搬你的凳子，我搬我的凳子。互不影响。"

三类传地址的数据：①引用 ②数组 ③对象。

5. 重点概念对比

概念 A 概念 B 区别
实例方法 类方法 无 static（对象调用） / 有 static（类名调用）
构造方法 普通方法 无返回值类型、名与类同名、自动调用 / 有返回值类型、手动调用
重载（Overload） 重写（Override） 同一类内、方法名同参数不同 / 父子类之间、方法签名完全相同
值传递 址传递 基本类型传副本互不影响 / 引用类型传地址一处变处处变
实参 形参 调用时传的具体值 / 定义时的占位变量

6. 代码示例整理

示例 1：构造方法重载

public class Person {
    String name;
    int age;

    // 构造方法1：无参（缺省）
    public Person() {
        name = "未命名";
        age = 0;
    }

    // 构造方法2：一个参数（重载）
    public Person(String name) {
        this.name = name;
        this.age = 0;
    }

    // 构造方法3：两个参数（重载）
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void show() {
        System.out.println("姓名：" + name + "，年龄：" + age);
    }
}

public class Main {
    public static void main(String[] args) {
        Person p1 = new Person();               // 调用构造方法1
        Person p2 = new Person("Tom");          // 调用构造方法2
        Person p3 = new Person("Alice", 20);    // 调用构造方法3

        p1.show();  // 姓名：未命名，年龄：0
        p2.show();  // 姓名：Tom，年龄：0
        p3.show();  // 姓名：Alice，年龄：20
    }
}

示例 2：实例方法与类方法的区别

public class Counter {
    int instanceCount = 0;        // 实例变量
    static int classCount = 0;    // 类变量（static）

    // 实例方法
    public void addInstance() {
        instanceCount++;
    }

    // 类方法
    public static void addClass() {
        classCount++;
        // instanceCount++;  ← 错误！类方法不能访问实例变量
    }
}

public class Main {
    public static void main(String[] args) {
        Counter.addClass();           // 类名调用类方法
        System.out.println(Counter.classCount);  // 1

        Counter c = new Counter();
        c.addInstance();              // 对象调用实例方法
        System.out.println(c.instanceCount);     // 1
    }
}

示例 3：值传递与址传递对比

public class PassDemo {
    // 值传递——基本类型
    public static void changeValue(int x) {
        x = 100;   // 只改了副本
    }

    // 址传递——数组（引用类型）
    public static void changeArray(int[] arr) {
        arr[0] = 100;  // 改了原数组！
    }

    public static void main(String[] args) {
        // 值传递：原值不变
        int a = 10;
        changeValue(a);
        System.out.println("值传递后 a = " + a);   // 10（没变）

        // 址传递：原数组被修改
        int[] arr = {10, 20, 30};
        changeArray(arr);
        System.out.println("址传递后 arr[0] = " + arr[0]);  // 100（变了！）
    }
}

7. 易错点总结

易错点 1：类方法（static）中访问实例变量
错误：在 static 方法中直接写 this.name 或直接访问非 static 变量——编译错误。
原因：类方法在对象创建前就存在，此时实例变量还没有内存空间。

易错点 2：构造方法写了返回值类型
错误：public void Person() { }——这已经不是构造方法了，变成普通方法了。
正确：public Person() { }——构造方法没有返回值类型，连 void 都不能写。

易错点 3：重载只改返回值类型不改参数
错误：两个方法都是 public void test() 和 public int test()——参数相同只改返回值类型不叫重载，编译错误！
正确：重载必须改参数列表（个数/类型/顺序）。

易错点 4：址传递中意外修改了原始数据
场景：把数组传给方法，在方法里修改了数组，方法外面数组也跟着变了——初学者常以为传的是副本。

8. 小测验题目

题目 1：以下哪个是构造方法的正确声明？
A. public void Person() { }
B. public Person() { }
C. public static Person() { }
D. public int Person() { }
正确答案：B（构造方法无返回值类型，连 void 也不能写）

题目 2：方法重载的"一同三不同"中，"三不同"指的是？
A. 返回值不同、方法名不同、参数不同
B. 参数个数不同、参数类型不同、参数顺序不同
C. 修饰符不同、返回值不同、方法体不同
D. 类不同、包不同、方法名不同
正确答案：B

题目 3：以下关于 static 方法的说法正确的是？
A. static 方法必须由对象调用
B. static 方法可以访问实例变量
C. static 方法属于类，由类名直接调用
D. static 方法中可以使用 this
正确答案：C

题目 4：值传递和址传递的区别是？
A. 值传递传地址，址传递传值
B. 值传递传副本（互不影响），址传递传地址（共享修改）
C. 两者没有区别
D. 值传递只用于对象，址传递只用于基本类型
正确答案：B

题目 5：以下代码的输出是？
int x = 5;
changeValue(x);
System.out.println(x);
// changeValue方法：void changeValue(int n) { n = 100; }

A. 5
B. 100
C. 编译错误
正确答案：A（int 是基本类型，值传递，传的是副本，原值不变）

9. 拖拽匹配素材

1. 实例方法 —— 无 static 修饰，必须由对象来调用
2. 类方法 —— 有 static 修饰，直接用类名调用
3. 构造方法 —— 与类同名、无返回值、创建对象时自动调用
4. 缺省构造方法 —— 程序员不写时系统自动生成的无参构造方法
5. 重载（Overload） —— 同一类中方法名相同、参数列表不同（一同三不同）
6. 值传递 —— 基本类型参数传递，传的是副本，互不影响
7. 址传递 —— 引用类型参数传递，传的是地址，一处变处处变
8. 形参 —— 方法定义时声明的占位变量
9. 实参 —— 方法调用时传入的具体数值
10. static —— 关键字，修饰的成员属于类而非对象

10. 课后总结

1. 本节课的核心是理解方法的两大分类（实例方法/类方法）、构造方法的特性与重载机制、以及参数传递的两种方式（值/址）。
2. 学习方法时要特别注意 static 的用法——实例方法/变量属于对象，类方法/变量属于类本身（所有对象共享）。
3. 重载是后续学习多态的基础——"一同三不同"（名称相同，个数/类型/顺序不同）是区分重载和其他概念的关键。
4. 值传递和址传递的区别会影响方法内部修改是否影响外部数据——基本类型安全，引用类型"一处变处处变"。
