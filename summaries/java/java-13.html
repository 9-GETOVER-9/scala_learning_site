5. 强调 Java 是单继承——一个子类有且只有一个直接父类，这是为避免"二义性"（两个父类有同名成员时无法判断来源）。
6. 引入多态的第一种形式——重写（Override）：发生在不同类体中，方法头相同（方法名、参数列表、返回值类型相同），方法体不同。

4. 知识点详细整理

知识点 1：四种访问权限的完整访问矩阵

这个知识点是什么？

四种访问权限修饰符控制类成员（属性和方法）在四种访问场景下的可见性：

访问场景 public protected default private
同一包下的同一个类 ✅ ✅ ✅ ✅
同一包下的不同类 ✅ ✅ ✅ ❌
不同包下的子类 ✅ ✅ ❌ ❌
不同包下的非子类 ✅ ❌ ❌ ❌
为什么学？

老师强调："我们的项目是在不同包下传递值的——你想要哪一个包的谁来看到谁，就用这四种修饰符来限制你所访问的变量，方法也是同理。"访问权限是面向对象封装的基础。

课堂中的理解方式

老师用四层递进的"财产"类比：

  • public（公有的）：寝室的水壶——谁都能用，全局可见
  • protected（受保护的）：家族财产（如房子）——有"血缘关系"的（同包 + 不同包子类）可以用，陌生人不行
  • default（缺省的/不写）：室友的东西——同寝室的可以借，但不是谁都能碰
  • private（私有的）：自己的银行卡——只有自己能用，谁也不能动，"用了钱就没了"
常见错误

以为 protected 比 default 只多了子类访问——实际上不同包下 default 连子类也访问不到。

知识点 2：继承的概念与生活类比

这个知识点是什么？

继承是一种由已有类创建新类的机制。利用继承，可以先创建一个共有属性的一般类（父类），再根据该类创建具有特殊属性和行为的新类（子类）。新类继承一般类的状态和行为。

为什么学？

避免代码重复——公有的属性和方法写在父类中，所有子类自动拥有，不需要每个子类重新写一遍。同时子类可以增加自己特有的功能，满足新的需求。

课堂中的理解方式

老师用"家族遗传"的比喻：

"你父亲浓眉大眼、天庭饱满、地阁方圆，那你很大几率就是这样——这叫继承属性。你父亲会唱歌、会跳舞、跑得很快，你也很大几率能继承下来——这叫继承方法。"

但继承不是全部照搬：

"你父亲只会唱红歌。你不仅会唱红歌，还会唱通俗、流行、摇滚、粤语、英语、法语——这是你产生的新方法。你既可以继承父亲的方法，也可以产生自己新的方法，这才属于你自己特殊的一个人。"

基本语法
// 父类（Super Class / Parent Class）
public class People {
    String name;
    int age;
    public void speak() {
        System.out.println("I can speak");
    }
}

// 子类（Sub Class / Child Class）
public class Student extends People {
    // Student 自动拥有 name、age、speak()
    // 还可以添加自己的新属性和新方法
    public void learnEnglish() {
        System.out.println("I can speak English too!");
    }
}

关键字：extends（扩展） 格式：class 子类名 extends 父类名

知识点 3：Java 的单继承原则

这个知识点是什么？

Java 采用单继承机制：一个子类有且只有一个直接父类。

为什么 Java 是单继承？

避免二义性（歧义性）。老师解释说：

"你想想，如果父亲 A 有个 name，父亲 B 也有个 name，那这个子类的 name 到底是从谁那继承来的？你就分不清了。这就叫二义性。"

C++ 多继承 vs Java 方案
语言 继承方式 如何解决二义性
C++ 多继承（一个子类可有多个父类） 用特殊语法（虚继承）处理
Java 单继承 用**接口（interface）**来补充多继承的需求

老师说明："在 Java 面向对象里面，只有一个父亲节点。兄弟节点可以多个，但父亲只有一个。……Java 为了也能产生类似多继承的情况，用了最后一章的接口（interface）来做这件事。"

生活中的类比

"这两个孩子只能有一个父亲。……C++ 就可以有多继承。但你想，如果家庭离异了，又找到了一个——但那是第二个，跟他是没有关系的，它的属性是继承不了的。"

知识点 4：继承的树形结构（倒置二叉树）

这个知识点是什么？

Java 类按照继承关系形成树形结构，像一棵倒置的二叉树——树根在上，枝叶在下。

        祖先（根节点 root）
         │
        父类（父亲节点）
        ┌┴┐
    子类A  子类B（兄弟节点）
     ┌┴┐
  孙A  孙B（叶子节点 leaf）
关键结论
  • 每一个子节点有且只有一个父节点（单继承）
  • 兄弟节点可以有多个
  • 从祖先到叶子，从上到下逐层创建
  • 最底层的节点如果没有子类，称为叶子节点
课堂中的理解方式

老师用"三代人"的族谱类比：

"这是爷爷，父亲继承了爷爷的东西。到了父亲这一代，两个儿子——他俩是兄弟。再往下，孙子辈——你看他俩长得挺像的，但是他们的子女长得就不太像了。但又有点像他小叔、像他小姨——因为有个共同的奶奶啊！"

知识点 5：重写（Override）——多态的第一种形式

这个知识点是什么？

重写（Override）是指子类定义一个与父类方法头完全相同但方法体不同的方法，从而隐藏父类的该方法。

重写的三个条件
条件 说明
发生位置 不同的类体中（父类和子类之间）
方法头 完全相同——方法名、参数列表、返回值类型都一样
方法体 不同——子类提供自己的实现
为什么学？

"子类的功能无法满足我的要求，我不想再继承你的了。我要满足我新的要求——你不擅长，我擅长。"

重写让子类能够在继承的基础上"改进"父类的行为，是多态的重要实现方式。

重写 vs 重载
比较维度 重写（Override） 重载（Overload）
发生位置 不同类体中（父子类之间） 同一个类体中
方法头 完全相同 方法名相同，参数列表不同
方法体 不同 不同（或相同）
典型例子 子类重新定义父类的 speak() 构造方法的不同参数版本
关键字 无需特殊关键字 无需特殊关键字
课堂代码演示
// 父类
public class People {
    public void speak() {
        System.out.println("I can speak Chinese");
    }
}

// 子类重写
public class Student extends People {
    @Override  // 注解，表示这是重写（可选但推荐）
    public void speak() {
        System.out.println("I like English!");  // 方法体不同
    }
}

当子类对象调用 speak() 时，执行的是子类重写后的版本，而不是父类的版本。

知识点 6：子类对象创建过程——先父后子

这个知识点是什么？

创建子类对象时，必须先创建父类对象，再创建子类对象。在构造方法的调用链上，子类构造方法总是先调用父类的某个构造方法。

为什么？

"你要想有，必须先有父亲。没有父亲，一定不会有你。"

从树形结构来看：创建对象时，必须从祖先节点开始，按层次从上到下依次创建，不能直接跳到子节点。

基本规则
  1. 如果父类有无参构造方法（系统自动提供或手动定义），子类构造方法会自动调用它
  2. 如果父类只有带参数的构造方法，子类必须在构造方法的第一句用 super(参数) 明确指定调用哪个父类构造方法
  3. 如果子类构造方法中没有写 super(...)，系统默认调用父类的无参构造方法
课堂代码演示
// 父类——只有带参构造方法
public class Father {
    String name;
    public Father(String name) {
        this.name = name;
        System.out.println("父类构造方法被调用");
    }
}

// 子类——必须用 super 明确调用父类构造方法
public class Son extends Father {
    public Son(String name) {
        super(name);  // 必须写在第一句！
        System.out.println("子类构造方法被调用");
    }
}

// 测试
public class Test {
    public static void main(String[] args) {
        Son s = new Son("小明");
        // 输出：
        // 父类构造方法被调用
        // 子类构造方法被调用
    }
}

5. 重点概念对比

概念 A 概念 B 区别
public private 全局可见 vs 仅本类可见
protected default 同包可见 + 不同包子类可见 vs 仅同包可见
继承 不继承（独立类） 子类自动拥有父类的非私有成员 vs 每个类从零开始
单继承（Java） 多继承（C++） 一个子类只有一个直接父类（避免二义性）vs 一个子类可有多个父类
重写（Override） 重载（Overload） 不同类体、方法头相同、方法体不同 vs 同一类体、方法名相同、参数列表不同
super this 代表父类对象的引用 vs 代表当前对象的引用
父类（Super Class） 子类（Sub Class） 被继承的类 vs 继承父类的类

6. 代码示例整理

示例 1：四种访问权限的完整演示

适用知识点：访问权限修饰符的四种访问场景

// 文件：com/example/Welcome.java（包 com.example）
package com.example;

public class Welcome {
    public    int pub = 1;     // 公有的
    protected int pro = 2;     // 受保护的
              int def = 3;     // 缺省的（default）
    private   int pri = 4;     // 私有的

    // 场景1：同一包下的同一个类——四个全可见
    public void showInSameClass() {
        System.out.println(pub);  // ✅
        System.out.println(pro);  // ✅
        System.out.println(def);  // ✅
        System.out.println(pri);  // ✅
    }
}

// 文件：com/example/First.java（同一包下的不同类）
package com.example;

public class First {
    public void accessWelcome() {
        Welcome w = new Welcome();
        System.out.println(w.pub);  // ✅ public——可见
        System.out.println(w.pro);  // ✅ protected——可见
        System.out.println(w.def);  // ✅ default——可见
        // System.out.println(w.pri);  // ❌ private——不可见！编译错误
    }
}

// 文件：unit/Tools.java（不同包下的子类）
package unit;                      // 注意：不同的包！
import com.example.Welcome;

public class Tools extends Welcome {
    public void accessFromSubclass() {
        System.out.println(pub);   // ✅ public——可见
        System.out.println(pro);   // ✅ protected——子类可见
        // System.out.println(def);  // ❌ default——不同包不可见
        // System.out.println(pri);  // ❌ private——不可见
    }
}

// 文件：unit/AfterPrint.java（不同包下的非子类）
package unit;
import com.example.Welcome;

public class AfterPrint {
    public void accessFromOutside() {
        Welcome w = new Welcome();
        System.out.println(w.pub);  // ✅ public——全局可见
        // System.out.println(w.pro);  // ❌ protected——非子类不可见
        // System.out.println(w.def);  // ❌ default——不同包不可见
        // System.out.println(w.pri);  // ❌ private——不可见
    }
}

运行结果：各场景的可见性严格按访问矩阵执行。

初学者提示：最容易搞混的是 protected vs default——记住"受保护的"有"血缘关系"的跨包特权（子类），而 default 只在同包范围内有效，跨包后连子类都看不见。

示例 2：继承的基本使用——子类继承父类的属性和方法

适用知识点：继承的概念、extends 关键字

// 父类：人
public class People {
    String name;
    int age;

    public People() {}                   // 无参构造方法

    public People(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void speak() {
        System.out.println(name + " can speak.");
    }

    public void run() {
        System.out.println(name + " can run.");
    }
}

// 子类：学生——什么都不用写就继承了父类的成员
public class Student extends People {
    // 空的！但已经拥有了 name、age、speak()、run()
}

// 测试类
public class Test {
    public static void main(String[] args) {
        Student s = new Student();
        s.name = "Charlie";
        s.age = 20;
        s.speak();  // 输出：Charlie can speak.
        s.run();    // 输出：Charlie can run.
    }
}

运行结果：

Charlie can speak.
Charlie can run.

解释：Student 类体中一行代码没写，但已经通过继承拥有了 People 的全部非私有成员。这就是继承的好处——公有代码不用重复写。

初学者提示：注意，Student 虽然什么都没有，但 new Student() 能成功，因为 People 有无参构造方法被自动调用。

示例 3：方法重写（Override）——子类改变父类的方法行为

适用知识点：重写（Override）、多态

// 父类
public class People {
    String name;
    
    public People(String name) {
        this.name = name;
    }
    
    public void speak() {
        System.out.println(name + " speaks Chinese.");
    }
}

// 子类——重写 speak() 方法
public class Student extends People {
    public Student(String name) {
        super(name);  // 调用父类构造方法
    }
    
    // 重写：方法头完全相同，方法体不同
    @Override
    public void speak() {
        System.out.println(name + " speaks English!");
    }
    
    // 子类还可以有自己的新方法
    public void learnFrench() {
        System.out.println(name + " is learning French.");
    }
}

// 测试类
public class Test {
    public static void main(String[] args) {
        People p = new People("父亲");
        p.speak();  // 输出：父亲 speaks Chinese.（父类自己的版本）
        
        Student s = new Student("查理");
        s.speak();  // 输出：查理 speaks English!（子类重写后的版本）
        s.learnFrench();  // 输出：查理 is learning French.（子类独有的方法）
    }
}

运行结果：

父亲 speaks Chinese.
查理 speaks English!
查理 is learning French.

解释：子类 Student 重写了 speak() 方法，所以当子类对象调用 speak() 时，执行的是子类的版本，而不是父类的版本。同时子类还增加了自己的新方法 learnFrench()——这就是"既可以继承，也可以扩展"。

初学者提示：

  • @Override 注解表示这是重写，它会让编译器帮你检查方法头是否与父类一致（推荐写）
  • 子类重写了父类的方法后，父类对象调用的还是父类自己的版本
  • 重写不能降低访问权限（父类是 public，子类不能改成 protected）

7. 课堂案例整理

案例 1：家族财产继承链

课堂场景：老师用三代人的家族关系来解释继承和访问权限。

爷爷（祖先）
 ├── 公有财产（public）：家族企业——谁都能看到
 ├── 受保护财产（protected）：房子、家具——只有家族成员能继承
 ├── 默认财产（default）：家族内部的东西——只有同族人能用
 └── 私有财产（private）：小金库——只有自己知道，谁都不能动

父亲继承爷爷 → 两个儿子继承父亲 → 孙子……

教学要点：

  • 父亲继承了爷爷的非私有财产——对应子类继承父类的非私有成员
  • 父亲的私有财产（小金库）谁也不能动——对应 private 成员不可继承
  • 两个儿子（兄弟节点）共享受保护的家族财产——对应 protected 在同包和子类中可见
  • 孙子既像父亲也像叔叔（共同的奶奶）——对应继承链中的"间接继承"

案例 2：重写——父子两代人的"唱歌"

课堂场景：用父亲和儿子的唱歌能力来解释重写。

"你父亲只会唱红歌——《井冈山》《洪湖赤卫队》。你继承了他的好嗓子，但你不仅能唱红歌，还能唱通俗、摇滚、粤语、英语、法语。你父亲不会的这些，你要产生一些新的方法，这才属于你自己特殊的一个人。"

映射到代码：

  • 父类的 sing() 方法只会"唱红歌"
  • 子类重写 sing() 方法，加入更多曲风（方法体扩展）
  • 子类还可以新增 singPop()、singRock() 等父类没有的方法

8. 易错点总结

易错点 1：以为子类能继承父类的一切——实际上 private 成员不可继承。父类的私有属性和方法只属于父类本身，子类也访问不到。如果需要访问，必须通过父类提供的 public 访问器（get/set）。

易错点 2：把 protected 和 default 的跨包行为搞反——protected 的不同包子类可以访问，default 跨包后连子类都看不见。口诀："protected 有血缘关系的能跨包，default 只认同一个包。"

易错点 3：在子类构造方法中忘记用 super 调用父类构造方法——当父类没有无参构造方法时，子类构造方法的第一句必须写 super(参数)，否则编译错误。

易错点 4：把重写（Override）和重载（Overload）混淆——重写发生在父子类之间（方法头完全相同），重载发生在同一个类中（方法名相同但参数列表不同）。

易错点 5：忘记 Java 是单继承——以为一个子类可以同时 extends 多个父类，这在 Java 中是语法错误。需要多继承的效果时，应使用接口（interface）来实现。

9. 小测验题目

题目 1：以下哪个访问权限修饰符修饰的成员，在"不同包下的子类"中也能访问？
A. private  B. default  C. protected  D. 只有 public
正确答案：C
解析：protected 允许不同包下的子类访问。default 跨包后子类也看不到。public 也可以，但题目问的是"也能"，protected 的最低要求就是跨包子类可访问。

题目 2：以下哪个关键字用于声明一个类继承另一个类？
A. implements  B. extends  C. super  D. import
正确答案：B
解析：extends 是继承关键字，格式为 class 子类 extends 父类。implements 是实现接口的关键字。

题目 3：Java 为什么采用单继承？
A. 为了代码更简洁  B. 为了运行速度更快  C. 为了避免二义性  D. 因为 JDK 限制
正确答案：C
解析：单继承避免了多个父类有同名成员时的歧义性（不知道从哪个父类继承）。多继承需求通过接口来补充。

题目 4：关于方法重写（Override），以下说法正确的是？
A. 重写可以发生在同一个类中  B. 重写时方法头可以不同  C. 重写时方法头相同、方法体不同  D. 重写必须使用 @Override 注解
正确答案：C
解析：重写的核心条件是方法头完全相同、方法体不同，发生在不同类体中（父子类之间）。@Override 注解推荐但非必须。

题目 5：子类对象创建时，构造方法的调用顺序是？
A. 先子类后父类  B. 先父类后子类  C. 只调用子类  D. 随机调用
正确答案：B
解析：子类构造方法总是先调用父类的某个构造方法（隐式或显式通过 super），先完成父类的初始化，再初始化子类。

10. 拖拽匹配素材

1. public —— 公有的，全局可见，所有包都能访问
2. protected —— 受保护的，同包和不同包子类可访问
3. default（不写） —— 缺省的，仅同一包下可访问
4. private —— 私有的，仅本类自身可访问
5. extends —— Java 继承关键字，用于声明父子类关系
6. 父类（Super Class） —— 被继承的类，提供共有的属性和方法
7. 子类（Sub Class） —— 继承父类的类，可以增加新的属性和方法
8. 单继承 —— Java 的继承机制，一个子类有且只有一个直接父类
9. 二义性 —— 多继承中无法判断成员来源的问题，Java 通过单继承避免
10. 重写（Override） —— 子类定义与父类方法头相同、方法体不同的方法

12. 课后练习题

练习题 1：写出四种访问权限修饰符在"同一包不同类"场景下的可见性（✅/❌）。
知识点：访问权限修饰符
参考答案：public ✅ / protected ✅ / default ✅ / private ❌
解题思路：同一包下，只有 private 是本类独有的，其他三个都能在同包的不同类中访问。

练习题 2：定义一个 Animal 父类（属性 name，方法 eat()），再定义一个 Cat 子类继承 Animal，并重写 eat() 方法输出"猫吃鱼"。
知识点：继承语法、方法重写
参考答案：
class Animal { String name; void eat() { System.out.println("动物吃东西"); } }
class Cat extends Animal { @Override void eat() { System.out.println("猫吃鱼"); } }
解题思路：使用 extends 建立继承关系，子类中写与父类方法头相同的 eat() 但方法体改为"猫吃鱼"。

练习题 3：解释"上转型对象"的含义。如果 Animal a = new Cat(); a.eat(); 会输出什么？为什么？
知识点：上转型对象、多态
参考答案：会输出"猫吃鱼"。虽然 a 的类型是 Animal（父类），但它实际引用的是 Cat 对象，调用方法时会调用 Cat 重写后的版本。这就是"父亲花儿子挣的钱"。
解题思路：Java 的方法调用看实际对象类型（运行时多态），不看引用变量的声明类型。

练习题 4：以下代码会编译错误吗？如果会，请说明原因并修正。
class Father { int age; Father(int age) { this.age = age; } }
class Son extends Father { String name; Son(String name) { this.name = name; } }
知识点：子类构造方法与 super
参考答案：会编译错误。父类没有无参构造方法，子类构造方法必须显式调用 super(参数)。修正：
class Son extends Father {
    String name;
    Son(int age, String name) {
        super(age);  // 第一句必须调用父类构造方法
        this.name = name;
    }
}
解题思路：当父类只有带参构造方法时，子类构造方法第一句必须写 super(参数)。

练习题 5：简述重写（Override）和重载（Overload）的四个区别。
知识点：多态的两种形式
参考答案：
① 发生位置：重写在父子类之间；重载在同一个类中。
② 方法头：重写方法头完全相同；重载方法名相同但参数列表不同。
③ 方法体：重写方法体不同；重载方法体可同可不同。
④ 目的：重写用于改变继承来的方法行为；重载用于提供同一功能的不同参数版本。
解题思路：记忆口诀——重写是"不同类、同头不同体"；重载是"同类、同名不同参"。

14. 课后总结

1. 本节课的核心是"两条线"：一条是访问权限的深入复习（四种修饰符 × 四种场景的完整矩阵），另一条是继承机制的正式引入（概念、语法、单继承、重写）。
2. 访问权限是封装的基础——通过限制访问范围来保护数据。记住四句口诀：public 水壶随便用，protected 家族内部用，default 室友一起用，private 自己单独用。
3. 继承的本质是"复用"——父类写好共有的东西，子类拿来就用，不用重复写。但继承不是"全盘照抄"——private 不继承，子类还可以重写（Override）不满意的方法。
4. Java 是单继承（一个子类一个爹），这是为了避免"两个爹争一个娃"的二义性问题。多继承需要通过接口实现。
5. 子类对象创建时"先父后子"——要创建儿子，必须先有父亲。父类有带参构造时，子类必须用 super(参数) 明确调用。
