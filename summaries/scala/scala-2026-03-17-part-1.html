4. 知识点详细整理

知识点 1：转置矩阵的概念

这个知识点是什么？

转置矩阵就是把矩阵的行和列进行交换。例如原来第 i 行第 j 列的元素，转置后会移动到第 j 行第 i 列。

为什么要学它？

矩阵转置在数据处理、线性代数、图像处理、机器学习等场景中都很常见。老师用它作为“二维遍历”的进阶例子。

基本语法

没有固定语法，这是一个算法/思维问题。

课堂中的理解方式

老师说：对于 3×3 矩阵，A11、A12、A13……这些元素在转置后会变成另一种位置。通过调换 i 和 j，就能实现转置。这个过程和二维数组的遍历密切相关。

初学者容易犯的错误
  1. 不知道转置后是“行列互换”。
  2. 把转置和倒序混淆。
  3. 不会用循环写出转置逻辑。
正确示例
object TransposeDemo {
  def main(args: Array[String]): Unit = {
    val a = Array(
      Array(1, 2, 3),
      Array(4, 5, 6),
      Array(7, 8, 9)
    )

    for (j <- 0 until 3) {
      for (i <- 0 until 3) {
        print(a(i)(j) + " ")
      }
      println()
    }
  }
}
示例解释
  • 原矩阵 a 的行和列分别是 3×3。
  • 外层遍历列 j，内层遍历行 i。
  • a(i)(j) 被以转置方式输出。

知识点 2：步长采样——按间隔取值

这个知识点是什么？

步长表示每次循环跳过多少个元素。Scala 中常用 by 指定步长，比如每 2 个取一个值、每 3 个取一个值。

为什么要学它？

老师把它和“采样”联系起来：在计算机科学和工程中，不一定需要每个数据点都看，只看间隔采样的点就足够。

基本语法
for (i <- 1 to 10 by 3) {
  println(i)
}
课堂中的理解方式

老师把步长类比成“走路一步跨多远”或者“物理里的频率/振幅概念”：你不是每次都走一步，而是按固定间隔跨步前进。比如 1 to 10 by 3 就是 1、4、7、10。

初学者容易犯的错误
  1. 把步长理解成“循环次数”。
  2. 不知道 by 是和范围配合使用的。
  3. 认为步长只能是正数。
正确示例
object StepSampling {
  def main(args: Array[String]): Unit = {
    for (i <- 1 to 10 by 2) {
      println(i)
    }
  }
}
示例解释
  • 从 1 开始。
  • 每次加 2。
  • 输出的是奇数序列。

知识点 3：Scala 函数的完整结构

这个知识点是什么？

Scala 函数通常由 def、函数名、参数列表、返回类型和函数体组成。

为什么要学它？

函数是 Scala 编程最重要的基本单位之一。你写的业务逻辑几乎都要封装成函数。

基本语法
def 函数名(参数名: 类型, 参数名: 类型): 返回类型 = {
  函数体
}
课堂中的理解方式

老师把函数拆成几部分解释：

  • def 是定义函数的关键字；
  • 函数名是这个功能的名字；
  • 参数列表表示输入；
  • 返回类型表示输出；
  • 函数体表示怎么做。

老师还提醒：Scala 的重点信息往前放，类型等信息往后放，这和 Java 的写法顺序正好相反。

初学者容易犯的错误
  1. 函数名和参数类型写反。
  2. 忘记写返回类型。
  3. 函数体最后一行和返回值关系不清楚。
正确示例
def add(x: Int, y: Int): Int = {
  x + y
}
示例解释
  • add 是函数名。
  • x: Int, y: Int 是参数列表。
  • : Int 是返回值类型。
  • x + y 是函数体最后一行，也是返回值。

知识点 4：递归思想

这个知识点是什么？

递归是指一个函数在内部调用自己，用来解决“结构相同、规模不同”的问题。

为什么要学它？

递归是很多算法的基础，如阶乘、斐波那契数列、树形结构遍历等。老师虽然没有完全展开，但已经把它列为本节课的重要内容。

基本语法
def factorial(n: Int): Int = {
  if (n <= 1) 1
  else n * factorial(n - 1)
}
课堂中的理解方式

老师说递归适合“规则相同但参数不同”的情况。也就是说，你只要会处理一个小问题，就能用相同规则去处理更大的问题。

初学者容易犯的错误
  1. 不写递归出口，导致无限递归。
  2. 不理解递归调用的“分解-回收”过程。
  3. 以为递归就是“循环换了个名字”。
正确示例
def factorial(n: Int): Int = {
  if (n <= 1) 1
  else n * factorial(n - 1)
}
示例解释
  • 当 n 小于等于 1 时，递归结束。
  • 否则不断调用自己处理更小规模的问题。

知识点 5：函数参数和返回值的意义

这个知识点是什么？

函数参数是输入，返回值是输出。Scala 中函数的参数写在函数名后面，类型放在参数名后面；返回类型写在参数列表后面。

为什么要学它？

理解参数和返回值，才能写出真正可复用的函数。

基本语法
def distance(x1: Int, y1: Int, x2: Int, y2: Int): Double = {
  math.sqrt((x2 - x1) * (x2 - x1) + (y2 - y1) * (y2 - y1))
}
课堂中的理解方式

老师把 Java 的方法调用流程和 Scala 函数做了对比：

  • Java 常常需要先定义对象，再调用方法，再传递参数；
  • Scala 可以更直接地把输入和输出放到函数里。
初学者容易犯的错误
  1. 不清楚参数和返回值的方向。
  2. 把参数名和参数类型写反。
  3. 不知道函数返回类型该写在哪里。
正确示例
def sum(a: Int, b: Int): Int = {
  a + b
}
示例解释
  • a、b 是输入参数。
  • Int 是输入和输出类型中的一部分。
  • 函数最终返回一个 Int 值。

知识点 6：变长参数（简单了解）

这个知识点是什么？

变长参数允许你在调用函数时传入不定数量的参数。Scala 使用 * 表示变长参数。

为什么要学它？

在处理大数据、批量输入、多个参数集合时非常方便。虽然课堂里没有深入展开，但它是 Scala 函数的重要特性之一。

基本语法
def printAll(nums: Int*): Unit = {
  nums.foreach(println)
}
课堂中的理解方式

老师说 Scala 在处理这类问题时比 Java 更简洁，大数据处理经常用到这种灵活参数形式。

初学者容易犯的错误
  1. 不知道 * 表示变长参数。
  2. 把变长参数和数组完全等同。
  3. 不会在调用时传多个值。
正确示例
def show(names: String*): Unit = {
  names.foreach(println)
}
示例解释
  • String* 表示可传入多个字符串。
  • foreach 遍历每个名字并输出。

6. 代码示例整理

示例 1：矩阵转置输出

适用知识点：

转置矩阵、二维遍历

代码：

object MatrixTranspose {
  def main(args: Array[String]): Unit = {
    val a = Array(
      Array(1, 2, 3),
      Array(4, 5, 6),
      Array(7, 8, 9)
    )

    for (j <- 0 until 3) {
      for (i <- 0 until 3) {
        print(a(i)(j) + " ")
      }
      println()
    }
  }
}

运行结果：

1 4 7 
2 5 8 
3 6 9

代码解释：

  1. 原矩阵按行定义。
  2. 输出时外层循环列，内层循环行。
  3. 通过交换 i 和 j 完成转置输出。

初学者提示：

转置的关键是把“行优先”变成“列优先”。

示例 2：步长采样

适用知识点：

步长、采样、by

代码：

object StepByDemo {
  def main(args: Array[String]): Unit = {
    for (i <- 1 to 10 by 3) {
      println(i)
    }
  }
}

运行结果：

1
4
7
10

代码解释：

  • 每次增加 3。
  • 这是典型的规律采样。

初学者提示：

by 是“步长”，不是“次数”。

示例 3：递归阶乘

适用知识点：

递归、条件分支、函数返回值

代码：

def factorial(n: Int): Int = {
  if (n <= 1) 1
  else n * factorial(n - 1)
}

object RecursionDemo {
  def main(args: Array[String]): Unit = {
    println(factorial(5))
  }
}

运行结果：

120

代码解释：

  1. 当 n<=1 时停止递归。
  2. 否则继续调用自己。
  3. 5! = 5×4×3×2×1 = 120。

初学者提示：

递归一定要有出口，否则会无限调用自己。

8. 易错点总结

易错点 1：转置时不交换行列

错误示例：
还是按原来的 a(i)(j) 顺序输出。

正确写法：
转置输出时要注意行列交换。

解释：
转置的核心就是 i/j 位置互换。

易错点 2：步长当成次数

错误示例：
1 to 10 by 3 只循环 3 次。

正确理解：
by 3 表示每次加 3，不是总共循环 3 次。

易错点 3：函数没有出口

错误示例：
def f(n: Int): Int = f(n - 1)

正确写法：
def f(n: Int): Int = if (n <= 1) 1 else f(n - 1)

解释：
递归必须有结束条件。

易错点 4：参数类型和参数名顺序写反

错误示例：
def add(Int a, Int b): Int = a + b

正确写法：
def add(a: Int, b: Int): Int = a + b

解释：
Scala 参数名在前，类型在后。

易错点 5：把变长参数当数组直接用

错误理解：
String* 就等于 String[]。

正确理解：
变长参数是函数参数写法的一种，不是普通数组。

12. 课后练习题

练习 1：写一个函数 sum，接收两个 Int 参数，返回它们的和。

考查知识点：def、参数列表、返回值

参考答案：

def sum(a: Int, b: Int): Int = {
  a + b
}

解题思路：函数接收两个整数，最后一行返回相加结果。

练习 2：用步长 2 输出 1 到 10。

考查知识点：for、by、范围

参考答案：

object StepTwo {
  def main(args: Array[String]): Unit = {
    for (i <- 1 to 10 by 2) {
      println(i)
    }
  }
}

解题思路：使用 by 2 控制步长。

练习 3：编写递归函数求 1 到 n 的累加和。

考查知识点：递归、结束条件

参考答案：

def sumToN(n: Int): Int = {
  if (n <= 1) 1
  else n + sumToN(n - 1)
}

解题思路：递归的出口是 n<=1，其他情况不断向下递归。

练习 4：写一个函数，接收多个字符串并依次输出。

考查知识点：变长参数、foreach

参考答案：

def show(names: String*): Unit = {
  names.foreach(println)
}

解题思路：String* 表示可传多个字符串。

练习 5：用二维循环输出一个 2×2 矩阵的坐标名，如 C11、C12、C21、C22。

考查知识点：二维遍历、for

参考答案：

object Matrix2x2 {
  def main(args: Array[String]): Unit = {
    for {
      i <- 1 to 2
      j <- 1 to 2
    } {
      println("C" + i + j)
    }
  }
}

解题思路：外层控制行，内层控制列。

  3. 通过本节课，你应该能看懂简单的递归函数，也能把二维矩阵的行列交换逻辑写出来。

  4. 变长参数是 Scala 函数的一个灵活特性，后面写工具函数、批量处理多个参数时会很常见。

  5. 课后建议多练习：把矩阵转置和步长采样自己在代码里写一遍，再尝试把递归函数（阶乘、斐波那契）改写出来。重复写几遍，才能真正理解函数的结构。
