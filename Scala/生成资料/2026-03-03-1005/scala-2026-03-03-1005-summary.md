## 一、课堂摘要

本节课主要围绕 Java 与 Scala 课程的学习要求、课程意义、AI 对未来就业的影响、课程安排、软件安装顺序、Scala 开发环境使用，以及 Scala 变量定义进行讲解。

## 二、核心知识点

### 1. 学习要求：必须多动手

老师反复强调，本课程不能只靠听。Scala 比 Java 更抽象，很多底层结构被封装起来，代码看起来短，但理解难度更高。

学习建议：

- 多敲代码。
- 不要只复制实验。
- 同一段代码建议反复敲两三遍。
- 训练输入速度。
- 先搭好环境，再上实验课。

---

### 2. Java 与 Scala 的课程意义

Java 和 Scala 都不是只为当前课程服务，而是为后续大数据、云计算、AI 等方向打基础。

课堂中提到：

- Java 与 Hadoop、大数据存储相关。
- Scala 与 Spark、内存计算相关。
- Spark 平台在大数据计算中非常重要。
- 学习语言的目的不只是会写代码，而是形成架构思想和软件开发能力。

### 3. AI 对学习和就业的影响

老师用多个例子说明 AI 已经深入很多行业，包括软件开发、数据分析、应急应用、军事策略等。

核心观点：

### 5. 软件安装顺序

老师强调软件安装有顺序要求。

大致要求：

1. 先安装第一个基础软件。
2. 配置好 JDK / GPT 等依赖。
3. 再安装第二个 Scala 相关软件。

本节课只是做初步介绍。

| Scala trait | Scala 特质，后续课程会重点讲 |

老师提到，Scala 中对象比 Java 中对象更难，后续会重点讲对象和单例对象。

---

### 8. var 与 val

本节课介绍了 Scala 中最基础的变量定义。

```scala
var i = 10
var j = 11
val k = i + j
```

解释：

- `var` 定义变量，值可以改变。
- `val` 定义常量，值不建议或不能重新赋值。
- `k` 的值是 `i + j`，结果为 `21`。
- Scala 中语句末尾可以省略分号。

---

### 9. Scala 尽量减少手动操作

老师强调 Scala 的一个特点：

> 尽可能减少程序员的手动操作。

例如：

- 分号可以省略。
- 很多 Java 中需要写的结构，在 Scala 中被封装。
- Scala 代码可能一两行完成 Java 十几行代码的效果。

但这也带来一个问题：

> 代码看起来简单，背后的思想更抽象。

## 三、代码示例

### 示例 1：定义变量和常量

```scala
var i = 10
var j = 11
val k = i + j
```

解释：

- `i` 和 `j` 是变量。
- `k` 是常量。
- `k` 的结果是 `21`。

---

### 示例 2：创建对象并访问属性

```scala
class Test {
  var i = 10
  var j = 11
  val k = i + j
}

object Main {
  def main(args: Array[String]): Unit = {
    val c = new Test
    println(c.k)
  }
}
```

解释：

- `Test` 是 Scala 类。
- `new Test` 创建对象。
- `c.k` 访问对象中的 `k`。
- 输出结果是 `21`。

## 四、小测验

### 题目 1

Scala 中定义变量使用哪个关键字？

A. val  
B. var  
C. final  
D. object  

**正确答案**：B  
**解析**：`var` 用来定义变量，值可以改变。

---

### 题目 2

Scala 中定义常量使用哪个关键字？

A. class  
B. var  
C. val  
D. package  

**正确答案**：C  
**解析**：`val` 用来定义不可变值。
