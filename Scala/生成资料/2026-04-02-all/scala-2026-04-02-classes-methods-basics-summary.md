## 一、课堂摘要

本节课主要讲解 Scala 编程中的循环、类、对象、方法和函数，并布置了一个面向对象相关的课堂作业。

老师首先强调循环的重要性。循环是解决重复动作的重要工具，很多项目、数据分析、样本处理和 AI 任务，本质上都需要根据某些条件反复执行同类动作。学习 Scala 时，不能只记语法，还要理解循环在解决问题中的作用。

随后课堂进入 Scala 类和对象的讲解。老师先用 Java 中的 `student class` 做对比，说明类可以理解为一种模板或模具，对象是根据类创建出来的具体实例。类本身是抽象的，不占用具体对象空间；对象是类的具体实现，占用内存空间。

课堂重点强调了 Scala 类相比 Java 类更抽象、更灵活。Scala 中可以通过 `object` 表示特殊对象，也可以通过单例对象、伴生对象等方式减少 Java 中可能出现的重复类和冗余结构。Scala 里“万物皆对象”，对象可以拥有自己的属性和方法。

本节课还讲解了 Scala 类的基本特点：一个源文件中可以包含多个类，类可以不用显式声明 `public`，类中可以定义变量和方法，也可以带类参数。类参数从定义处开始，在整个类体中都可以访问，作用域一直到类体结束。

最后，老师讲解了方法和函数的区别。方法是类的一部分，通常通过对象反复调用；函数可以理解为一个完整对象，常用于临时性、一次性的计算动作。课堂作业要求同学创建一个对象或类，例如 Animal、Dog、Cat，包含两个变量、一个方法和一个函数。

## 二、核心知识点

### 1. 循环的重要性

循环用于解决重复动作，例如：

- 重复处理样本；
- 重复读取数据；
- 重复判断条件；
- 重复执行计算；
- 重复输出结果。

在项目开发、大数据分析和 AI 任务中，循环都是非常重要的基础结构。

---

### 2. 循环依赖条件

循环通常不是盲目重复，而是根据条件进行重复。

例如：

```scala
for (i <- 1 to 10) {
  println(i)
}
```

这里循环条件是范围 `1 to 10`，程序会依次处理范围中的每一个值。

---

### 3. 类是对象的抽象

类可以理解为对象的模板。

例如：

```scala
class Student
```

`Student` 不是某一个具体学生，而是所有学生共同特征的抽象。

类可以包含：

- 属性；
- 方法；
- 类参数；
- 行为规则。

---

### 4. 对象是类的实例

对象是根据类创建出来的具体实例。

```scala
val s = new Student
```

这里 `s` 就是 `Student` 类创建出来的一个对象。

类是模板，对象是模板制造出来的具体东西。

---

### 5. 类本身不占具体对象空间

类更像设计图或模具，它描述对象应该有什么属性和方法。只有真正创建对象时，才会为对象分配具体空间。

这和 Java 中类与对象的思想类似。

---

### 6. Scala 类比 Java 类更抽象

Java 中通常通过创建多个类来表达不同角色和特殊任务。Scala 可以用更灵活的方式，比如 `object`、单例对象和伴生对象，减少冗余结构。

例如：

- 普通学生可以由 `Student` 类表示；
- 特殊角色可以用对象或伴生对象表达；
- 避免为了每个特殊情况都创建额外类。

---

### 7. 万物皆对象

Scala 强调“万物皆对象”。

对象可以拥有：

- 属性；
- 方法；
- 自己独立的行为；
- 与其他对象交互的能力。

这比 Java 中“对象主要用于调用类成员”的理解更灵活。

---

### 8. 单例对象

单例对象可以理解为整个程序中只有一个实例的对象。

```scala
object School {
  val name = "Scala School"
}
```

这里 `School` 是对象，不需要 `new` 就可以直接使用。

---

### 9. 伴生对象

如果一个 `class` 和一个 `object` 同名，它们可以形成伴生关系。

```scala
class Student {
  var name = "Tom"
}

object Student {
  def create(): Student = new Student
}
```

伴生对象可以配合类完成一些特殊创建或管理任务。

---

### 10. 一个 Scala 文件可以包含多个类

Scala 中一个源文件可以包含多个类，也可以包含多个对象。

这和 Java 不完全一样。Java 通常一个 public 类对应一个文件，而 Scala 更灵活。

---

### 11. Scala 类默认更偏 public

Scala 中类不显式声明 `public` 时，通常可以先理解为公有可见。

这也是 Scala 写法比 Java 简洁的地方之一。

---

### 12. 类可以带参数

Scala 类名后可以直接带参数。

```scala
class Point(xc: Int, yc: Int) {
  var x = xc
  var y = yc
}
```

这里 `xc` 和 `yc` 就是类参数。

---

### 13. 类参数作用域

类参数从定义处开始，在整个类体内部都可以访问。

```scala
class Point(xc: Int, yc: Int) {
  var x = xc
  var y = yc

  def move(dx: Int, dy: Int): Unit = {
    x = x + dx
    y = y + dy
  }
}
```

`xc` 和 `yc` 可以在类内部使用。

---

### 14. 类中的方法

方法是类的一部分，通常描述对象可以做什么。

```scala
class Person {
  def speak(): Unit = {
    println("Hello")
  }
}
```

方法可以通过对象调用。

```scala
val p = new Person
p.speak()
```

---

### 15. 方法与函数的区别

课堂中老师强调：

- 方法是类的一部分；
- 函数可以理解为对象；
- 方法常用于对象需要重复调用的行为；
- 函数常用于临时性、一次性的计算。

语义上两者相近，但使用场景和所属位置不同。

---

### 16. 匿名函数

函数可以没有名字，直接表示一个动作。

```scala
val f = (x: Int) => x + 1
```

这里 `f` 保存了一个函数，输入一个整数，返回这个整数加 1 的结果。

---

### 17. 方法定义结构

方法通常使用 `def` 定义。

```scala
def add(a: Int, b: Int): Int = {
  a + b
}
```

组成部分包括：

- `def` 关键字；
- 方法名；
- 参数列表；
- 返回值类型；
- 等号；
- 方法体。

---

### 18. 没有等号的方法

如果方法没有等号，可能被理解为没有明确返回值，或者在特定语境下成为抽象方法声明。

学习时要注意：

- 有等号：强调返回表达式的结果；
- 无等号：通常不返回有意义的值，或者只是方法头；
- 不要随意省略等号。

---

### 19. 对象调用方法

如果方法属于类，就可以通过对象调用。

```scala
val p = new Person
p.speak()
```

这种调用方式和 Java 中“对象.方法名”基本类似。

## 三、代码示例

### 示例 1：创建简单类和对象

```scala
class Student {
  var name = "Tom"
  var age = 20
}

val s = new Student
println(s.name)
println(s.age)
```

解释：`Student` 是类，`s` 是通过 `new Student` 创建的对象。

---

### 示例 2：带参数的类

```scala
class Point(xc: Int, yc: Int) {
  var x = xc
  var y = yc
}

val p = new Point(1, 2)
println(p.x)
println(p.y)
```

解释：`xc` 和 `yc` 是类参数，用来初始化对象属性。

---

### 示例 3：类中的方法

```scala
class Point(xc: Int, yc: Int) {
  var x = xc
  var y = yc

  def move(dx: Int, dy: Int): Unit = {
    x = x + dx
    y = y + dy
  }
}
```

解释：`move` 是类的方法，可以改变对象的坐标。

---

### 示例 4：对象调用方法

```scala
val p = new Point(1, 2)
p.move(3, 4)

println(p.x)
println(p.y)
```

解释：对象 `p` 调用 `move` 方法后，坐标发生变化。

---

### 示例 5：匿名函数

```scala
val addOne = (x: Int) => x + 1

println(addOne(5))
```

解释：`addOne` 是一个函数对象，接收整数并返回加 1 后的结果。

---

### 示例 6：作业示例 Animal

```scala
class Animal {
  var name = "dog"
  var age = 2

  def run(): Unit = {
    println(name + " is running")
  }

  val speak = () => println(name + " is speaking")
}
```

解释：这个类包含两个变量、一个方法和一个函数，符合课堂作业要求。

## 四、小测验

### 题目 1

类和对象的关系是什么？

A. 类是对象的抽象，对象是类的实例  
B. 对象是文件，类是图片  
C. 类只能输出字符串  
D. 对象不能有属性  

**正确答案**：A  
**解析**：类像模板，对象是根据模板创建出来的具体实例。

---

### 题目 2

Scala 中创建对象常用哪个关键字？

A. new  
B. delete  
C. import  
D. package  

**正确答案**：A  
**解析**：可以使用 `new 类名` 创建对象。

---

### 题目 3

方法通常属于哪里？

A. 类的一部分  
B. 压缩包的一部分  
C. 图片的一部分  
D. 文件名的一部分  

**正确答案**：A  
**解析**：方法通常定义在类中，是类的一部分。

---

### 题目 4

Scala 中定义方法使用哪个关键字？

A. def  
B. var  
C. val  
D. objectOnly  

**正确答案**：A  
**解析**：Scala 使用 `def` 定义方法。

### 题目 5

下面哪个更像匿名函数？

A. `(x: Int) => x + 1`  
B. `class Student`  
C. `new Point`  
D. `package test`  

**正确答案**：A  
**解析**：匿名函数没有函数名，只描述参数和函数体。

## 六、流程动画建议

### 流程 1：类到对象的创建过程

```text
定义一个 class
↓
在类中写属性和方法
↓
使用 new 创建对象
↓
对象获得类中定义的属性
↓
对象可以调用类中定义的方法
↓
程序根据对象完成具体任务
```

### 流程 2：方法调用过程

```text
创建对象
↓
对象名后面写点号
↓
调用方法名
↓
传入需要的参数
↓
执行方法体
↓
对象状态或输出结果发生变化
```
