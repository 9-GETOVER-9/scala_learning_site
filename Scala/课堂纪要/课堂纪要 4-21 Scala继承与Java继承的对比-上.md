Scala 继承与 Java 继承的对比
2026-04-21  10:05
【智能总结】
本次课堂主要对比 Scala 与 Java 的继承，包括参数传递、Override 使用、继承特性及语法等方面的差异。
- **参数传递**：只有主步骤函数（主构造器）能向基类（父类）的步骤函数写参数，辅助步骤方法基于主步骤函数产生。
- **Override 使用规则**：
    - **Scala 重写规则**：重写超类的抽象方法时，可不写Override；重写非抽象方法必须写Override。
    - **Java 对比**：Java 中重写方法使用@Override注解，Scala 与之在写法上有差异。
- **继承特性**：
    - **共性**：Scala 和 Java 中，子类都继承父类非私有、非静态成员，可增加自身成员，且子类访问权限≥父类。
    - **Scala 特性**：Scala 只允许继承一个父类；final修饰的类、方法、字段，分别不可被继承、重写、修改；protected修饰的成员在同一包下可见可继承；子类重写父类var修饰的属性和方法需用Override。
- **语法差异**：
    - **定义类**：Java 类定义在构造方法里带参数列表，Scala 的主构造器直接带参数列表。
    - **构造对象**：Java 构造对象时按匹配的构造函数传参，Scala 构造对象时主构造器代码从头到尾执行一遍。
- **重写字段**：
    - **VAL 字段重写**：子类可用同名的VAL字段重写父类的VAL字段，且必须写Override。
    - **VAR 字段重写**：子类只能重写父类的var属性。
